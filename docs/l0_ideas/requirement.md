# L0 要求ドキュメント — dアニメストア ニコニコ支店ビューア

> これは **要求（欲求・背景・判断基準）の大元**。コードもフェーズ分けも書かない。
> 以降の L1（要件）・L2（構成）・L3（フェーズ）・実装は、すべてこの文書に照らして判断する。

## 背景・痛み

- ユーザーは dアニメストア **ニコニコ支店** で日常的にアニメを観ているが、**公式サイトの UI が見づらく**、
  「観たいもの・観るべきものに出会う」までの導線が弱い。
- ニコニコの**公開 API** を使えば、支店のラインナップやメタデータ（再生数・タグ・ジャンル・投稿時間など）は取得できる。
- そこで、**自分にとって見やすい「視聴／発見サイト」**を自前で作りたい。重厚なプラットフォームではなく、
  GitHub Pages に置けるような**軽い静的サイト**でよい。

## 北極星（最重要の判断基準）

> **「最小驚きの原則 ＋ ユーザー主権」**

- ランキングも検索も並び替えも、**「こう動くだろう」という予測どおりに動く**こと。
  根拠とする原理: **Principle of Least Astonishment（最小驚きの原則）／ Nielsen のユーザビリティ・ヒューリスティクス／
  Hick の法則（選択肢過多は意思決定を遅らせる）**。
- **アテンション経済の逆を行く。** YouTube 的な「滞在時間最適化・不透明なアルゴリズム・ダークパターン・自動再生」を**しない**。
  ユーザーの時間を奪うのではなく、**素早く目的に到達させて離脱させる**ことを良しとする。
- これは“プラットフォーム”ではなく **“自分のための道具”**。データも挙動もユーザーの側にある（ユーザー主権）。
- **「見やすさ」は主観で決めない。** 行動心理・認知科学の原理で根拠づける（上記ヒューリスティクスに紐づける）。

## 欲しい体験（欲求）

- **ランキング**で人気がひと目で分かる。特に**急上昇**（今 伸びているもの）を見たい。
- **タグ**で分かりやすく辿れる。さらに**お気に入りタグから新しい作品を開拓**できる。
- **近いジャンル／近い作品**を提示してくれる。賢い**リコメンド**で発見が広がる。
- **クールごとに遡れる**（今季だけでなく過去季も時間軸で探索できる）。
- 作品・シリーズを**快適にブラウズ**できる（網羅性と回遊のしやすさ）。
- **視聴は公式プレイヤーへ**送る（本ツールは発見に特化し、視聴体験は公式に委ねる）。

## スコープ厳守

- 対象は **ニコニコ支店のみ**。
- dアニメストア **本店（NTT DOCOMO, animestore.docomo.ne.jp）は別サービス**＝
  **ラインナップも視聴プレイヤーも別、対象外**。本店データを混ぜない。

## 将来像（要求としてのみ記載。ここでは実装しない）

- **ローカルファーストな視聴履歴連携**（ActivityWatch 型）で「続きから／全話視聴済み」などの
  パーソナライズを実現したい。データはユーザーの手元に置く（ユーザー主権）。＝**後段**。
- **急上昇**は、API が時系列を返さないため、`viewCounter` を**日次スナップショットして差分（伸び率）**で算出する。＝**後段**。

## この文書の使い方（判断基準として）

新しい機能・設計・UI を検討するときは、次を自問する:

1. それは**予測どおり**に動くか（最小驚き／Hick の法則に反していないか）。
2. それは**ユーザーの時間を奪う**方向（アテンション経済）に寄っていないか。
3. それは**ニコニコ支店のみ**のスコープを守っているか。
4. 「見やすさ」の根拠を**行動心理の原理**で説明できるか。
