# L0 要求ドキュメント — dアニメストア ニコニコ支店ビューア

> これは **要求（欲求・背景・判断基準）の大元**。コードもフェーズ分けも書かない。
> 以降の L1（要件）・L2（構成）・L3（フェーズ）・実装は、すべてこの文書に照らして判断する。

## 背景・痛み

- ユーザーは dアニメストア **ニコニコ支店** で日常的にアニメを観ているが、**公式サイトの UI が見づらく**、
  「観たいもの・観るべきものに出会う」までの導線が弱い。
- ニコニコの**公開 API** を使えば、支店のラインナップやメタデータ（再生数・タグ・ジャンル・投稿時間など）は取得できる。
- そこで、**自分にとって見やすい発見サイト**を自前で作りたい。重厚なプラットフォームではなく、
  GitHub Pages に置けるような**軽い静的サイト**でよい。

## 大事にしたいこと（設計の指針）

- **予測どおりに動く。** 同じ操作をすれば同じ結果になる。ランキングや並び替えに**隠れた重み付けをしない**。
- **時間を奪わない。** 自動再生・無限スクロールの罠・ダークパターンをしない。**素早く目的に到達させて離脱できる**ようにする。
- **選択肢を絞って迷わせない。** 一画面に詰め込みすぎず、探し方（ランキング/タグ/ジャンル/クール）を分かりやすく分ける。
- **見やすさは思いつきで決めない。** 「なぜ見やすいか」を説明できる形にする（上の3点に沿っているか）。

## 欲しい体験（欲求）

- **ランキング**で人気がひと目で分かる。
- **タグ**で分かりやすく辿れる。
- **ジャンル**で気分から探せる。
- **クールごとに遡れる**（今季だけでなく過去季も時間軸で探索できる）。
- 作品・シリーズを**快適にブラウズ**できる（網羅性と回遊のしやすさ）。
- **視聴は公式プレイヤーへ**送る（本ツールは発見に特化し、視聴体験は公式に委ねる）。

## スコープ厳守

- 対象は **ニコニコ支店のみ**。
- dアニメストア **本店（NTT DOCOMO, animestore.docomo.ne.jp）は別サービス**＝
  **ラインナップも視聴プレイヤーも別、対象外**。本店データを混ぜない。

## やらないこと / 将来（v1 スコープ外）

- **急上昇**（今 伸びている作品）は、API が時系列を返さないため `viewCounter` を**日次で記録して差分**を出す必要がある。
  仕組みの追加が要るので**後段**で扱う。
- **近い作品の提示（リコメンド）**は、タグ等の類似で根拠が説明できる形なら将来検討するが、**v1 では作らない**。
- **視聴履歴を使った個人化**（続きから／全話視聴済み 等）は、**視聴履歴を集める別プロジェクト（ブラウザ拡張など）に依存する**。
  本ビューアの対象外で、その別プロジェクトができてからの話。**v1 では扱わない。**

## この文書の使い方（判断基準として）

新しい機能・設計・UI を検討するときは、次を自問する:

1. それは**予測どおり**に動くか（同じ操作で同じ結果。隠れた並び替えをしていないか）。
2. それは**ユーザーの時間を奪わない**か（自動再生・罠のような導線になっていないか）。
3. それは**ニコニコ支店のみ**のスコープを守っているか。
4. 「見やすい」と言える**理由を説明できる**か。
