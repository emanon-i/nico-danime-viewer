# PH-0004: 探索機能（検索・五十音・タグ・クール・並び替え・導線）

## 目的
一覧を探索の主戦場として機能させる。検索／五十音／フラットタグ／クール絞り／並び替え／クイックアクセス・プリセット（TOP10・新着2種）／パンくずを実装し、トップ・一覧・詳細の回遊が予測どおりに成立する状態にする。screens.md 準拠。

## 機能一覧

### F-0027: 検索（全画面共通）
**対応REQ**: REQ-0001（screens.md 検索仕様）

ヘッダ🔍は押すと検索バーが展開・自動フォーカス、Esc/× で閉じる。トップ＝ヒーロー検索（主役）、一覧＝最上部バー（維持）、詳細＝ヘッダ🔍 のみ。入力確定で**一覧の検索結果（`?q=…`）**へ集約。画面に入力欄が同時に 2 本出ない。

**受け入れ条件**:
- [ ] 🔍 で検索バーが展開しフォーカス、Esc/× で閉じる
  - 検証: テスト `test_search_expand_collapse`
- [ ] どの画面で打っても `?q=…` で一覧の検索結果へ遷移する
  - 検証: テスト `test_search_routes_to_list`
- [ ] 同一画面に入力欄が同時に2本出ない（トップはヒーロー隠れ後にヘッダ🔍）
  - 検証: テスト `test_single_search_input`

---

### F-0028: 五十音ナビ
**対応REQ**: REQ-0011（screens.md §2, validation A）

五十音ボタン（あ〜わ）で `col_key` の行に絞る（`?row=…`）。並び替え「五十音順」（`?sort=kana`）は行（あ→わ）順、**行内はタイトル文字列順フォールバック**（完全 yomi が無いため厳密 50 音順は不可）。

**受け入れ条件**:
- [ ] 五十音ボタンで該当行の作品に絞られる（`?row=…`）
  - 検証: テスト `test_kana_row_filter`
- [ ] 「五十音順」ソートが行順＋行内 title 順で決定的に並ぶ
  - 検証: テスト `test_kana_sort_row_then_title`
- [ ] 行内の厳密 50 音順を主張する UI を出さない（注記/フォールバックを正直に）
  - 検証: テスト/目視不要（実装上 yomi ソートを行わない）

---

### F-0029: タグ絞り（フラット1系統）
**対応REQ**: REQ-0003 / REQ-0008（screens.md §2/§1, foundation §1.2）

一覧の「タグ」フィルタは**フラット 1 系統**（`?tag=…`）。キュレーション正規化タグも通常タグと同列（別 facet にしない・候補の優先表示はしてよい）。トップの「タグから探す」（Hot/人気/ランダム/定番）チップは click で `?tag=…` の一覧へ送る。

**受け入れ条件**:
- [ ] タグ選択で `?tag=…` により絞り込まれる（複数タグ併用可なら AND を明示）
  - 検証: テスト `test_tag_filter`
- [ ] キュレーションタグが通常タグと同じフラット集合で扱われる（別 facet が無い）
  - 検証: テスト `test_tags_are_flat`
- [ ] トップのタグチップ（Hot/人気/ランダム/定番）が `?tag=…` の一覧へ遷移する
  - 検証: テスト `test_top_tag_chip_navigates`
- [ ] ランダムタグの [🔀] で別サンプルに引き直せる
  - 検証: テスト `test_random_tag_reshuffle`

---

### F-0030: クール絞り
**対応REQ**: REQ-0004（screens.md §1/§2, validation C）

クールボタン（`?cours=…`）で放送季に絞る。今期は現行季プリセット、過去季は period 由来。判定不能（`cours=NULL`）は「クール不明」で正直に扱い、絞り対象から外す/明示する。

**受け入れ条件**:
- [ ] クール選択で `?cours=…` により該当季に絞られる
  - 検証: テスト `test_cours_filter`
- [ ] 「今期」プリセットが現行季＋勢い順で開く
  - 検証: テスト `test_current_cours_preset`
- [ ] `cours=NULL` は「クール不明」として扱い、季の集計に混ぜない
  - 検証: テスト `test_cours_unknown_handling`

---

### F-0031: 並び替え
**対応REQ**: REQ-0002（screens.md ランキング方針）

並び替えは**勢い順／再生数(累計)順／新着順／五十音順**。すべて決定的（同操作→同結果・隠れた重み付けなし）。**人気TOP に期間セレクタを置かない**（履歴非蓄積で月間/週間は作れない）。

**受け入れ条件**:
- [ ] 4 種の並び（勢い/累計/新着/五十音）が選べ、決定的に並ぶ
  - 検証: テスト `test_sort_options_deterministic`
- [ ] 期間セレクタ（週間/月間）を UI に出さない
  - 検証: テスト `test_no_period_selector`
- [ ] フィルタ（タグ/クール/五十音）と並びが併用できる
  - 検証: テスト `test_filter_sort_combination`

---

### F-0032: クイックアクセス・プリセット（TOP10・新着2種）
**対応REQ**: REQ-0002/0004/0010/0013（screens.md §1 プリセット表/新着の2種）

トップの **[今期][新着][Hot][人気TOP]** が一覧の特定プリセット URL を開く（今期=`?cours=現行季&sort=hot`、新着=新着シリーズ順、Hot=勢い順、人気TOP=累計順）。人気シリーズ TOP10 ショーケース（累計順・有限帯）。新着は**新着シリーズ（シリーズ初出）と最新の動画（話単位）の2系統**。

**受け入れ条件**:
- [ ] 4 プリセットが screens.md の表どおりの URL 状態を開く
  - 検証: テスト `test_quick_access_presets`
- [ ] TOP10 が累計再生数順の上位10件を表示する
  - 検証: テスト `test_top10_by_total_views`
- [ ] 新着が「新着シリーズ（min start_time）」「最新の動画（max start_time）」の2系統で出る
  - 検証: テスト `test_new_two_kinds`

### F-0033: パンくず
**対応REQ**: REQ-0001（screens.md パンくず仕様）

トップ以外はヘッダ直下にパンくずを表示し、**適用中の状態を反映**（`ホーム / 一覧：タグ「日常」` 等）。各セグメントは該当階層へクリック可、URL クエリ状態と整合。モバイルは先頭省略「…」。ヘッダに別途の「← 戻る」は置かない。

**受け入れ条件**:
- [ ] 一覧/詳細でパンくずが状態を反映して描画される
  - 検証: テスト `test_breadcrumb_reflects_state`
- [ ] 各セグメントのクリックで該当 URL 状態へ遷移する
  - 検証: テスト `test_breadcrumb_navigation`
- [ ] モバイルで長いパンくずは先頭省略「…」になる
  - 検証: テスト `test_breadcrumb_truncate_mobile`
