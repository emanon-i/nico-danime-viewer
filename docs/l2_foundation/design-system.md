## 0. 設計原則（このシステムが守ること）

| 原則                                     | 具体                                                                                                                                                 |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **画像が主役・UI は黒子**                | 色はサムネに任せる＝UI は無彩（チャコール/オフホワイト）＋寒色アクセント1色のみ。カードは**サムネ全面**、文字は最小限を画像下端にだけ重ねる。        |
| **余白に意味を持たせる（近接＝関係性）** | 余白は恣意的に決めない。**スペーシングスケール（トークン）とグリッドにだけ従う**。近い要素は関係が強い、離れた要素は関係が弱い、を一貫させる（§4）。 |
| **言語の圧縮（メタはアイコン＋数値）**   | 「全12話」「30万 再生」のような文字ラベルは出さず、**アイコン＋数値**に圧縮する（§8）。emoji は使わず Lucide バンドル SVG。                          |
| **異なるものは異なる見た目に**           | 役割が違うものは形・ラベル・メタの種類で区別する（例: シリーズ行 vs 各話行＝§9.3）。同じ見た目で別物を出さない。                                     |
| **定義より用途**                         | トークンは「何に・どの状態で使うか」まで必ず併記。曖昧なトークンは置かない。                                                                         |
| **DRY＝単一ソース**                      | カード/行/チップ/アイコンボタン/見出し/レールは Factory（`components/`）＋共通 CSS クラス1系統。インラインスタイル禁止（CSP）。                      |
| **予測どおり**                           | ホバーで隠す要素は可（操作アイコン）だがタッチ端末では常時表示にフォールバック。レイアウトシフトを出さない（`aspect-ratio`＋`width/height`）。       |

## 1. 対象・適用スコープ・CSP

| 項目           | 内容                                                                                                                 |
| -------------- | -------------------------------------------------------------------------------------------------------------------- |
| 対象           | `web/src/` の静的フロント（vanilla TS・独自 router・Vite）                                                           |
| CSS            | `web/src/style.css`（`@layer` 単一ファイル）                                                                         |
| コンポーネント | `web/src/components/`（`card.ts` / `listRow.ts` / `chip.ts` / `icon.ts`）                                            |
| FOUC 防止      | `web/public/theme-init.js`（`<head>` 同期ロード）                                                                    |
| 対象ブラウザ   | Chrome 112+ / Firefox 117+ / Safari 16.5+（CSS nesting・OKLCH・`color-mix`・`:has()`・subgrid はフォールバック付き） |

### 1.1 CSP（厳格・維持）

`web/index.html` の `<meta http-equiv="Content-Security-Policy">`：

```
default-src 'self'; script-src 'self'; style-src 'self'; connect-src 'self';
img-src 'self' https://nicovideo.cdn.nimg.jp https://*.nimg.jp;
base-uri 'self'; object-src 'none'
```

- `style-src 'self'` → **インライン `style=""` 禁止**。色・寸法はすべて `style.css` のクラス/トークンで表現。
- `script-src 'self'` → **インライン `<script>` 禁止**。`theme-init.js` も外部ファイルを同期ロード。
- `img-src` → サムネ実ホスト（§11）を許可済み。**追加変更は不要**。
- アイコンは **runtime fetch しない**（バンドル SVG）。`img-src` を増やさない（§8）。

## 2. 視覚方針（トーン＆マナー）

| 観点                                 | 方針                                                                                                                                                   |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **基調**                             | **ダーク基調**（チャコール）。サムネの色を最大限引き立てる暗い無彩面。                                                                                 |
| **サムネ主役**                       | カードは**サムネ全面**。UI は彩度を持たず、画面の色はほぼサムネ由来になるようにする。                                                                  |
| **無彩 UI ＋単一アクセント**         | UI 面・文字・境界は無彩（微青み色相 260・極低彩度）。**寒色アクセント（azure）1 色だけ**を限定使用（§6.3）。多色・カラーブロックは使わない。           |
| **ライトテーマ＝反転（純白でない）** | ライトは bg＝**オフホワイト**（`--p-ln-50`・純白でない）、カード操作面のみ純白可。陰影は弱める。ダークの設計を反転した同一構造（別デザインにしない）。 |
| **動きは控えめ**                     | ホバーの浮き・画像の微ズーム・テーマ切替のクロスフェード程度。自動再生・派手なアニメーションはしない（L0「時間を奪わない」）。                         |

## 3. グリッド＆スペーシング（重要・新設）

> **原則: 余白に意味を持たせる＝近接は関係性。** すべての余白はトークンとグリッドに従い、「なぜこの間隔か」を説明できること。

### 3.1 スペーシングスケール（4px 基底トークン）

| トークン | 値   | **用途（この間隔をどこで使うか）**                                  |
| -------- | ---- | ------------------------------------------------------------------- |
| `--sp-1` | 4px  | アイコンと数値の隙間・バッジ内パディング・微調整                    |
| `--sp-2` | 8px  | チップ/ボタン間隔・カード操作アイコンの余白・**強く関係する要素間** |
| `--sp-3` | 12px | カード内パディング・list-row の内側・行内ガター                     |
| `--sp-4` | 16px | セクション**見出しと中身**の間・要素グループ内の標準間隔            |
| `--sp-5` | 24px | セクション内ブロック間・カードレールのアイテム間                    |
| `--sp-6` | 32px | （予備）ブロック大間隔                                              |
| `--sp-7` | 48px | **セクション間の縦リズム**（標準）。隣接セクションの境              |
| `--sp-8` | 64px | 最大余白（必要時のみ）                                              |

- **恣意的な px を直書きしない**。新しい間隔が要るならまずスケールで表現できないか検討する。
- **近接ルール**：関係が強い要素ほど小さい間隔（`--sp-2`〜`--sp-3`）、独立したセクション同士は大きい間隔（`--sp-7`）。

### 3.2 コンテナ・ガター・整列

| トークン        | 値                       | 用途                                                       |
| --------------- | ------------------------ | ---------------------------------------------------------- |
| `--content-max` | `1280px`                 | 全セクションの**中身の最大幅**。これを超えて間延びさせない |
| `--gutter`      | `clamp(1rem, 5vw, 4rem)` | 画面左右の最小余白（流体）。全セクション共通               |

- **全セクションは同一の `--content-max` ＋ `--gutter` に揃える**。見出し・レール・リスト・チップ群・ボタン列の**左端・右端・カラム位置が画面全体で一致**すること（セクションごとに幅や余白を変えない）。
- ワイド画面では中身を `max-width` で中央寄せし、左右に `--gutter`。ウルトラワイドでも中身が散らからない。
- **横スクロール帯（カードレール）も左端＝`--gutter` から開始**し、他セクションの左端と揃える（`scroll-padding-inline: var(--gutter)`）。

### 3.3 セクション間の縦リズム

- セクション間は **`--sp-7`（48px）を標準**とし、必要に応じてセクション内パディング `clamp(--sp-5, 4vw, --sp-7)`。
- セクション境の区切り線（`border-bottom`）は**本文ショーケース（④以降）にのみ**引く。**ヒーロー検索とクイックアクセスの間には引かない**（両者は強く関係する入口同士＝§3.4）。

### 3.4 既知の是正点（近接で関係を表す）

- **ヒーロー検索 ↔ クイックアクセスは近接させる**：両者は「探し始める」入口として関係が強い。現状は縦の空きが大きすぎて関係が見えない → **間隔を詰める**（ヒーロー下パディングを小さく、罫線なしで連続させる）。検索 → クイックアクセス → 看板（TOP10）の順に、関係の強さに応じて間隔を段階化する。
- クイックアクセスは**ヒーロー検索の直下に左寄せ**で置き、検索バーの左端と揃える。

## 4. タイポグラフィ

### 4.1 フォント

**IBM Plex Sans JP**（Fontsource 自己ホスト・400/500/700）。`style.css` 先頭で `@import '@fontsource/ibm-plex-sans-jp/{400,500,700}.css'`。unicode-range 分割で日本語グリフは必要時のみ DL。数値は `font-variant-numeric: tabular-nums`（再生数・順位・件数・話数の桁揃え）。**日本語に italic は使わない**。

### 4.2 4段スケール（隣接段の差を一目で）

| 段     | トークン    | 値                                 | **この段だけが担う役割**                                                                               |
| ------ | ----------- | ---------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **L1** | `--text-l1` | `clamp(1.75rem, 4vw, 2.5rem)`      | **ヒーロー見出しのみ**（※現状トップはヒーロー見出しを置かない方針＝§9。L1 は他ページの「顔」用に温存） |
| **L2** | `--text-l2` | `clamp(1.125rem, 2.5vw, 1.375rem)` | **セクション見出し `h2`**・詳細の作品名・モーダル見出し                                                |
| **L3** | `--text-l3` | `clamp(0.875rem, 1.5vw, 1rem)`     | **本文・カード/行タイトル・入力・主要ボタン文字**。可読の基準                                          |
| **L4** | `--text-l4` | `clamp(0.6875rem, 1.2vw, 0.75rem)` | **メタ**（再生数・話数・投稿時間・件数・タグ・小見出し・パンくず・すべて見る）                         |

> 隣接段で**最低 1.25 倍**の差を保ち、階層を視覚的に即読み取れるようにする。中間段を増やさない（曖昧化を防ぐ）。

### 4.3 ウェイトルール（400/500/700）

| ウェイト                | 用途                                                                                                  | 禁止                       |
| ----------------------- | ----------------------------------------------------------------------------------------------------- | -------------------------- |
| **400 本文**            | あらすじ・各話/カードの**メタ数値**・入力テキスト                                                     | 見出しに使わない           |
| **500 ラベル/タイトル** | カード/行タイトル・ボタン文字・アクティブな並び替えラベル・ナビリンク・**各話行の投稿時間（強調用）** | —                          |
| **700 見出し/ランク**   | セクション `h2`・ロゴ・**ランク数字**・プライマリボタン・小見出しラベル・バッジ                       | 本文に使わない（うるさい） |

## 5. カラートークン（OKLCH・3層）

色は **primitive（パレット定数）→ semantic（用途）→ component（参照）** の3層。コンポーネントは semantic のみ参照。

### 5.1 プリミティブ（`--p-*`）

| 変数                        | 値                                              | 役割                                           |
| --------------------------- | ----------------------------------------------- | ---------------------------------------------- |
| `--p-ch-950`                | `oklch(0.16 0.005 260)`                         | チャコール最深（ダーク bg）                    |
| `--p-ch-900`                | `oklch(0.21 0.005 260)`                         | カード/パネル面                                |
| `--p-ch-850`                | `oklch(0.25 0.006 260)`                         | ホバー/画像欠損面                              |
| `--p-ch-700`                | `oklch(0.30 0.005 260)`                         | ボーダー                                       |
| `--p-ch-400`                | `oklch(0.60 0.005 260)`                         | ミュートテキスト                               |
| `--p-ch-200`                | `oklch(0.82 0.005 260)`                         | セカンダリテキスト                             |
| `--p-ch-100`                | `oklch(0.90 0.005 260)`                         | プライマリテキスト                             |
| `--p-az-dk` / `--p-az-dk-h` | `oklch(0.65 0.13 240)` / `oklch(0.72 0.13 240)` | azure アクセント（ダーク）/ホバー              |
| `--p-az-lt` / `--p-az-lt-h` | `oklch(0.48 0.16 245)` / `oklch(0.42 0.16 245)` | azure アクセント（ライト）/ホバー              |
| `--p-ln-50/0/100/200`       | `0.97 / 1.00 / 0.95 / 0.88`（彩度≈0・色相260）  | リネン bg / 純白カード面 / ホバー面 / ボーダー |
| `--p-sl-900` / `--p-sl-500` | `oklch(0.18 …)` / `oklch(0.50 …)`               | スレート プライマリ/ミュート文字（ライト）     |

> 純グレーでなく**色相 260（微青み）・彩度 0.002〜0.006**を全面に入れ、安っぽさを避けつつサムネ色を邪魔しない。

### 5.2 セマンティック（用途・状態つき）

| トークン                      | ダーク                               | ライト                     | **何に・どの状態で**                                                                  |
| ----------------------------- | ------------------------------------ | -------------------------- | ------------------------------------------------------------------------------------- |
| `--bg`                        | `--p-ch-950`                         | `--p-ln-50`                | ページ背景・sticky ヘッダ背景（半透明 blur）                                          |
| `--surface`                   | `--p-ch-900`                         | `--p-ln-0`                 | 入力/チップ/ボタン/パネルの**面**（カードは画像が面なので使わない）                   |
| `--surface-2`                 | `--p-ch-850`                         | `--p-ln-100`               | ホバー面・**サムネ欠損プレースホルダ面**・アイコンボタンホバー・各話バッジ面          |
| `--text`                      | `--p-ch-100`                         | `--p-sl-900`               | プライマリ文字（見出し・本文・ボタン）                                                |
| `--text-muted`                | `--p-ch-400`                         | `--p-sl-500`               | 副次文字・プレースホルダ・小見出し・ⓘ・**メタのアイコン＋数値**・非アクティブアイコン |
| `--border`                    | `--p-ch-700`                         | `--p-ln-200`               | 区切り線・入力/チップ/ボタンの輪郭（**装飾の色付けには使わない**）                    |
| `--accent`                    | `--p-az-dk`                          | `--p-az-lt`                | §6.3 の限定用途のみ                                                                   |
| `--accent-hover`              | `--p-az-dk-h`                        | `--p-az-lt-h`              | アクセント要素の `:hover`                                                             |
| `--focus-ring`                | `oklch(0.65 0.13 240/0.5)`           | `oklch(0.48 0.16 245/0.4)` | `:focus-visible`・入力フォーカス glow                                                 |
| `--shadow-sm` / `--shadow-md` | `0 1px 3px /0.4` / `0 4px 12px /0.5` | `…/0.1` / `…/0.12`         | 静止時の浮き / ホバー・モーダル                                                       |

**画像オーバーレイ専用の固定色**（テーマ非依存）：scrim＝`oklch(0 0 0/α)`、白文字＝`oklch(0.97〜1 0 0)`。画像上のコントラストはテーマで変えないため semantic を通さない。

### 5.3 アクセント・ポリシー（azure 1色）

- **使用限定**：リンク文字／アクティブ状態（タブ・ページ・♥✓ ON・シリーズバッジ）／`:focus-visible`／プライマリボタン（詳細の「公式」）／ロゴのマーク／ヒーロー背景のごく淡い radial（14% 透過）／メタアイコンの hover。
- **禁止**：装飾的ベタ塗り・カラーブロック・通常ボーダー（→`--border`）。
- **コントラスト**：azure はダーク/ライト各背景で文字対比 ≥ 4.5:1（WCAG AA）。`--accent` 上の文字は `--bg`。

## 6. テーマ機構

### 6.1 `data-theme` 3 状態

| 状態                 | 機構                            | 効果       |
| -------------------- | ------------------------------- | ---------- |
| 属性なし             | `@media (prefers-color-scheme)` | OS 追従    |
| `data-theme="dark"`  | `[data-theme="dark"]`           | 強制ダーク |
| `data-theme="light"` | `[data-theme="light"]`          | 強制ライト |

`<html>` に `color-scheme: dark light` を宣言し、ネイティブ UI（スクロールバー・フォーム）も追従。

### 6.2 FOUC 対策（`theme-init.js`）

`<head>` で**同期ロード**（`async`/`defer` 不可）。body 描画前に `localStorage('nico-danime-theme')` を読んで `data-theme` を確定。`script-src 'self'` 準拠の外部ファイル。

### 6.3 切替アニメーション

`theme.ts` の `applyTheme()` が `document.startViewTransition`（あれば）で切替。CSS は `@supports (view-transition-name: none)` ガードで `::view-transition-old/new(root){ animation-duration:0.2s }`。未対応は即時切替（劣化なし）。

### 6.4 ヘッダのアイコン順

右上は左→右で **`🔍 → ☀/🌙（テーマ）→ ⚙（設定/情報）`**。慣例どおり「頻用トグル＝内側、設定＝右端」。詳細動線は `screens.md`。

## 8. アイコン＆メタのアイコン圧縮

### 8.1 アイコン方針（Lucide バンドル SVG）

`components/icon.ts` に Lucide パスをハードコード。`icon(name, size?)` / `iconSvg(name, size?)`。**runtime fetch なし**（`img-src` 拡張不要・CSP 準拠）。`<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">`。**emoji は使わない**（描画差・CSP/一貫性のため）。ライセンス ISC（§14）。

| name              | 用途                                                                      |
| ----------------- | ------------------------------------------------------------------------- |
| `search`          | ヘッダ🔍・ヒーロー検索欄                                                  |
| `settings`        | ⚙ 設定/情報                                                               |
| `sun` / `moon`    | テーマトグル（実効テーマで出し分け）                                      |
| `heart` / `check` | ♥お気に入り / ✓見た（カード左上・詳細）                                   |
| `external-link`   | ↗ 公式ページ（カード右上・各 list-row）                                   |
| `shuffle`         | タグ「ランダム」の引き直し                                                |
| `info`            | ⓘ ツールチップ（各画面1か所）                                             |
| `x`               | モーダル/検索を閉じる                                                     |
| `chevron-right`   | 「すべて見る →」等                                                        |
| `play`            | **再生数メタの ▶**・各話サムネの ▶ オーバーレイ・サムネ欠損プレースホルダ |
| `film`            | **話数メタ（全N話・TOP10の話数）**                                        |
| `clock`           | **投稿時間メタ（各話行）**                                                |
| `layers`          | 新着シリーズ行の「シリーズ」種別バッジ                                    |

> 追加が必要なアイコンは Lucide から**パスをコピーして同梱**（CDN 参照しない）。

### 8.2 メタのアイコン圧縮（文字ラベルを置かない）

メタ情報は「全12話」「30万 再生」のような**語を書かず、アイコン＋数値**に圧縮する。共通パターン `.meta`：

```
.meta            … inline-flex・gap var(--sp-1)・色 --text-muted・L4・tabular-nums
.meta svg        … 0.85em 角・currentColor（数値と同色・同サイズ感）
複数メタは .meta を中黒なしで gap var(--sp-3) で並べる
```

| 意味     | 表現               | 例                                               |
| -------- | ------------------ | ------------------------------------------------ |
| 再生数   | `[play] 数値`      | `▶ 30万` / `▶ 132`                               |
| 話数     | `[film] 数値話`    | `▢ 12話`（全話数。単位「話」は残し「全」は省く） |
| 投稿時間 | `[clock] 相対時間` | `🕐 3時間前` / `🕐 6/16`                         |

- 数値整形：再生数は概数（`308万` / `9,876`）、話数は実数、投稿時間は相対（§9.3）。
- **アクセシビリティ**：アイコンは `aria-hidden`。意味は `aria-label`/`title` で各 `.meta` に与える（例 `aria-label="再生数 30万"`）。視覚は圧縮、支援技術には言語で補う。

## 9. コンポーネント仕様（DRY・状態つき）

### 9.0 形状・影・レイヤー

- **角丸**：`--r-sm`=4（チップ/バッジ）/ `--r-md`=8（カード・入力・ボタン）/ `--r-lg`=12（モーダル）/ `--r-full`（ピル・アイコンボタン・タグ）。
- **影**：`--shadow-sm`（静止）/ `--shadow-md`（ホバー・モーダル）。
- **z-index**：`--z-card-overlay`=1 / `--z-header`=10 / `--z-modal`=100。

### 9.1 ポスターカード `card()`（中核・TOP10／一覧／関連）

**Factory**：`card(seriesId, title, thumbnailUrl, officialHref?, meta?)`、`meta?: { rank?: number; views?: number; episodeCount?: number }`。

**DOM 契約**：

```
.series-card[data-series-id]              ← container-type: inline-size, aspect-ratio var(--card-aspect)=16/9, overflow hidden
  a.card-body[href="?series=<id>"]        ← 主＝うちの詳細。inset:0 全面
    img.card-img                          ← .L サムネ・object-fit:cover
    .card-overlay                         ← 下端 scrim（文字裏だけ暗く）・pointer-events:none
      span.card-rank?                     ← meta.rank 時のみ。大きな順位数字（cqw 連動・700）
      .card-text
        .card-title                       ← 2行 line-clamp・weight 500
        .card-meta?                       ← アイコン圧縮メタ（§8.2）：[play]再生数 ＋ [film]話数
  .card-actions                           ← 左上：♥/✓（オーバーレイ）
  a.card-external[target=_blank rel=noopener]  ← 右上：↗ 公式 /series（副）
```

**視覚仕様**：

| 項目         | 仕様                                                                                                                                                     |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 比率         | `aspect-ratio: var(--card-aspect)` ＝ **16/9**（§9.5）                                                                                                   |
| 画像         | `object-fit: cover`・ホバー `scale(1.04)`（0.4s）。欠損は `.no-thumb`（§9.4）                                                                            |
| scrim        | `linear-gradient(to top, 0.85 → 0.6@38% → 0)`。画像上部は暗くしない・`pointer-events:none`                                                               |
| ランク       | `clamp(1.5rem, 9cqw, 2.25rem)` 700 tabular。`@container`（カード幅）連動                                                                                 |
| **メタ**     | **[play] 再生数 ＋ [film] 話数**（§8.2）。**人気TOP10 も話数を必ず出す**（再生数だけにしない）                                                           |
| 操作アイコン | 32×32・`oklch(0 0 0/0.42)`＋`backdrop-filter blur(4px)`。既定透明→`:hover`/`:focus-within`表示。`@media (hover:none)`常時。`.active`は accent fill＋常時 |
| 当たり判定   | ♥✓＝左上 / ↗＝右上 を別位置（誤タップ防止）                                                                                                              |

**状態**：`:hover`/`:focus-within`＝`translateY(-3px)`＋`--shadow-md`＋画像 `scale(1.04)`＋操作アイコン出現／`.active`＝accent fill 常時／`.no-thumb`＝無地＋ラベル／loading＝サムネ面は `--surface-2`（`width/height` 指定済みでシフトなし）。

### 9.2 カードレール `.card-rail`（有限ランキング帯）

`display:flex; overflow-x:auto; scroll-snap-type:x proximity`。**スクロールバー非表示**。両端 `mask-image` フェードで「続きあり」を示す。`scroll-padding-inline: var(--gutter)`（左端を他セクションと揃える＝§3.2）。

### 9.3 list-row（シリーズ行／各話行＝2型・差別化）

`components/listRow.ts`。**ポスターカードとは別物**。「最近追加・更新」で使用。root=`div`、`a.list-row-body`（主）＋`a.list-row-external`（↗ 副・常時表示）で**アンカーをネストさせない**。サムネは `.L`・16:9 cover（`--card-aspect`）、欠損は **▶ プレースホルダ**（`play`）。

**`data-kind` で 2 型を明確に差別化（メタの種類で語る）**：

|              | **シリーズ行** `data-kind="series"` | **各話行** `data-kind="episode"`                          |
| ------------ | ----------------------------------- | --------------------------------------------------------- |
| 何を語るか   | **シリーズ＝話数で語る**            | **各話＝新しさ（投稿時間）で語る**                        |
| バッジ       | `layers`＋「シリーズ」（accent 面） | 「第N話」（surface 面）                                   |
| サムネ       | 16:9                                | 16:9 ＋ **▶ オーバーレイ**（単一動画の合図）              |
| タイトル     | 作品名                              | 話タイトル（**バッジと重複する「第N話」は本文から除去**） |
| メタ（§8.2） | **[film] 全N話**                    | **[clock] 投稿時間（強調）** ＋ [play] 再生数             |
| 主アクション | うちの詳細 `?series=<id>`           | 公式 watch（外部）                                        |
| 副 ↗         | 公式シリーズ `/series/<id>`         | 公式 watch `/watch/<contentId>`                           |

- **「第N話」はタイトル表記を優先**（公式表記と一致）。タイトルに無ければ nvapi `episode_no` をフォールバック（番号がずれることがあるため後順）。
- **投稿時間の強調**：各話の「新しさ」が主役。相対表記（`たった今`／`N分前`／`N時間前`／`N日前`、7日超は `M/D`、年跨ぎは `YYYY/M/D`）を `--text`／weight 500 で他メタより目立たせる。
- **整列**：2列（左=シリーズ／右=各話）は横並び時、**件数を揃え（最大5）・見出し同高・各行同高**で行ベースラインを揃える（§10.2）。モバイルは縦積み。

### 9.4 サムネ欠損・ローディング

- カード：`thumbnailUrl` null（実測 96 件）→ `.no-thumb`。`img` を `visibility:hidden`、`.card-body::before` に「NO IMAGE」を `--surface-2` 面で表示。
- list-row：サムネ無し（未解決の最新話等）→ **▶ プレースホルダ**（無地でなく意味のある代替）。
- ローディング：`aspect-ratio`＋`width/height` 指定でシフトを出さない。面は `--surface-2`。`loading="lazy"` / `decoding="async"`。

### 9.5 カード比率の決定（16:9・黒帯トリミング）

- 素サムネ（`.L` 360×270）は **4:3 枠に 16:9 映像を上下黒帯でレターボックスした焼き込み**（実画像で確認＝§11）。当初の「4:3＝全フレーム表示」は**前提が誤り**で、実際は黒帯を表示していた。
- **16:9 を採用**：`aspect-ratio: var(--card-aspect)`（=`16/9`）＋`object-fit:cover` で上下黒帯（各 1/8＝33.75px）を**正確にトリミング**、実コンテンツのみ全面表示（コンテンツ欠けなし）。
- 比率は **`--card-aspect` 1 トークン**で全箇所追従（`.series-card`／`.list-row-thumb`）。
- 留意：真に 4:3（黒帯なし）素材は上下がわずかに切れるが、大多数の 16:9 レターボックスを正しく扱うことを優先。

## 10. セクション別レイアウト方針（トップ）

> 順序・動線の正本は `screens.md`。ここでは**見た目に関わる方針**（大小・整列・列挙）だけを定める。

### 10.1 人気シリーズ TOP10（看板・主役）

- **他セクションより明確に大きいカード**でメリハリを出す：`.card-rail .series-card { flex: 0 0 clamp(264px, 32vw, 360px) }`（モバイル `flex-basis: 72vw`＝次がチラ見え）。list-row（小）との大小差で「看板」と分かる。
- **メタは [play]再生数 ＋ [film]話数**（再生数だけにしない＝§9.1）。
- PC/UW でも横スクロール可・バー非表示・続きは「すべて見る →」。

### 10.2 最近追加・更新（2列・整列・差別化）

- 左＝**新着シリーズ（シリーズ行）**／右＝**最新の動画（各話行）**。§9.3 の 2 型で**別見え**に。
- **2 列整列**：件数を揃え（最大5）、見出し同高、各行同高で**行ベースラインを揃える**（subgrid 可・非対応は固定高フォールバック）。モバイルは 1 列に縦積み。

### 10.3 クールから探す（全季を列挙する）

**現状の事実（なぜ少ないか）**：`cours.json` は **1 件（2026-春）のみ**。原因は **(c) 過去季ソース未配線**：

- `fetch.mjs` は今季の `programlist.json` だけを `mapCurrentCours()` に通し、**今季シリーズの `series.cours` しか埋めない**。
- 過去季の `period` HTML パーサ（`scripts/etl/cours.mjs` の `parsePeriodHtml` 等）は**存在するが `fetch.mjs` に未配線**（過去季ループ呼び出しが無い）。
- そのため `series.cours` は今季以外 null → `cours.json` は今季 1 件だけになる。
- **`episode.startTime` はクール判定に一切使われていない**（クールは programlist/period のタイトル突合由来）。

**表示方針（是正）**：全エピソードに `startTime` がある（フル取得済み）ので、**各シリーズの初出 `startTime`（=各話 startTime の最小）から「年＋季」を機械的に導出**し、**カタログに存在する全クールを新しい順に列挙**する（2026春・2026冬・2025秋・2025夏 …）。各ボタン → 一覧のクール絞り（`?cours=…`）。

- **季の境界**：`冬=1–3 / 春=4–6 / 夏=7–9 / 秋=10–12`（`filter.ts currentCoursLabel()` と一致）。ラベルは `YYYY-季`（例 `2026-春`）。
- 直近数季はインライン、それ以前は「過去季 ▾」に畳む。
- period（公式クール表）由来の**正確な帰属は将来の精緻化**でよい。まずは **startTime 導出で全季を出し、「1 季しか出ない」を解消**する。
- データ生成（startTime → 年+季の導出・`cours.json` 化・`fetch.mjs` への配線）の仕様は `foundation.md` / `dataflow.md` を正本とし、本書は**表示として全季を列挙する方針**を定める。

```ts
export function hiResThumb(url: string | null): string {
  if (!url) return ''
  return /\/thumbnails\/\d+\/\d+\.\d+$/.test(url) ? `${url}.L` : url
}
```

> 旧プレビューでサムネが出なかったのは CSP/ホスト/データではなく、**スクショが画像ロードを待たず撮影していた**こと（＋lazy）。検証は `complete && naturalWidth>0` を待つ（§16）。
