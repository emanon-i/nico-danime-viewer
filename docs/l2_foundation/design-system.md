## 0. 設計原則（このシステムが守ること）

| 原則                                           | 具体                                                                                                                                                                                                             |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **画像が主役・UI は黒子**                      | 色はサムネに任せる＝UI は無彩（チャコール/オフホワイト）＋寒色アクセント1色のみ。カードは**サムネ全面**、文字は最小限を画像下端にだけ重ねる。                                                                    |
| **余白に意味を持たせる（近接＝関係性）**       | 余白は恣意的に決めない。**スペーシングスケール（トークン）とグリッドにだけ従う**。近い要素は関係が強い、離れた要素は関係が弱い、を一貫させる（§4）。区切りは罫線でなく**余白**で表す（§3.3）。                   |
| **言語の圧縮（メタはアイコン＋最小の単位語）** | 冗長なラベルは出さず、**アイコン＋最小の単位語**（`[film]12話`・`[clock]3時間前`・`[play]30万`）に圧縮する（§8）。純アイコンのみは不可（支援技術には `aria-label` で補う）。emoji は使わず Lucide バンドル SVG。 |
| **異なるものは異なる見た目に**                 | 役割が違うものは形・ラベル・メタの種類で区別する（例: シリーズ行 vs 各話行＝§9.3）。同じ見た目で別物を出さない。                                                                                                 |
| **定義より用途**                               | トークンは「何に・どの状態で使うか」まで必ず併記。曖昧なトークンは置かない。                                                                                                                                     |
| **DRY＝単一ソース**                            | カード/行/チップ/アイコンボタン/見出し/レール/ツールチップ/段階表示は Factory（`components/`）＋共通 CSS クラス1系統。インラインスタイル禁止（CSP）。                                                            |
| **予測どおり**                                 | ホバーで隠す要素は可（操作アイコン）だがタッチ端末では常時表示にフォールバック。レイアウトシフトを出さない（`aspect-ratio`＋`width/height`）。`prefers-reduced-motion` で動きを止める（§13/§17）。               |

## 1. 対象・適用スコープ・CSP

> **実装状況** 【実装済】 — CSP は `web/index.html` に適用済み。`web/public/theme-init.js` FOUC 防止実装済み。

| 項目           | 内容                                                                                                                                                              |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 対象           | `web/src/` の静的フロント（vanilla TS・独自 router・Vite）                                                                                                        |
| CSS            | `web/src/style.css`（`@layer` 単一ファイル）                                                                                                                      |
| コンポーネント | `web/src/components/`（`card.ts` / `listRow.ts` / `chip.ts` / `icon.ts` / `meta.ts` / `marquee.ts` / `reveal.ts` / `tooltip.ts`）                                 |
| FOUC 防止      | `web/public/theme-init.js`（`<head>` 同期ロード）                                                                                                                 |
| フォント       | `@fontsource/ibm-plex-sans-jp` を**実 woff2 ファイルで self-host**（Vite `assetsInlineLimit:0` で `data:` インライン化を無効化＝厳格 CSP の `font-src` を満たす） |
| 対象ブラウザ   | Chrome 112+ / Firefox 117+ / Safari 16.5+（CSS nesting・OKLCH・`color-mix`・`:has()`・subgrid はフォールバック付き）                                              |

### 1.1 CSP（厳格・維持）

`web/index.html` の `<meta http-equiv="Content-Security-Policy">`：

```
default-src 'self'; script-src 'self'; style-src 'self'; font-src 'self'; connect-src 'self';
img-src 'self' https://nicovideo.cdn.nimg.jp https://*.nimg.jp;
base-uri 'self'; object-src 'none'
```

- `style-src 'self'` → **インライン `style=""` 禁止**。色・寸法はすべて `style.css` のクラス/トークンで表現。動的位置（ツールチップ）は CSSOM（`element.style` プロパティ代入）で指定し、インライン属性文字列は使わない。
- `script-src 'self'` → **インライン `<script>` 禁止**。`theme-init.js` も外部ファイルを同期ロード。
- `font-src 'self'` → フォントは self-host の woff2 のみ（`data:` フォントは弾く）。Vite のインライン化を無効化して全 woff2 を self ファイルにする。
- `img-src` → サムネ実ホスト（§11）を許可済み。
- `object-src 'none'` → プラグイン埋め込みを禁止。
- アイコンは **runtime fetch しない**（バンドル SVG）。`img-src` を増やさない（§8）。

## 2. 視覚方針（トーン＆マナー）

> **実装状況** 【実装済】 — ダーク/ライトテーマ・OKLCH トークン・カード scrim（下端グラデ）・ホバーズーム・操作アイコン背景 blur すべて実装済み。

| 観点                                 | 方針                                                                                                                                                                             |
| ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **基調**                             | **ダーク基調**（チャコール）。サムネの色を最大限引き立てる暗い無彩面。                                                                                                           |
| **サムネ主役**                       | カードは**サムネ全面**。UI は彩度を持たず、画面の色はほぼサムネ由来になるようにする。                                                                                            |
| **無彩 UI ＋単一アクセント**         | UI 面・文字・境界は無彩（微青み色相 260・極低彩度）。**寒色アクセント（azure）1 色だけ**を限定使用（§5.3）。多色・カラーブロックは使わない。                                     |
| **ライトテーマ＝反転（純白でない）** | ライトは bg＝**オフホワイト**（`--p-ln-50`・純白でない）、カード操作面のみ純白可。陰影は弱める。ダークの設計を反転した同一構造（別デザインにしない）。                           |
| **動きは控えめ**                     | ホバーの浮き・画像の微ズーム・テーマ切替のクロスフェード程度。自動再生・派手なアニメーションはしない（L0「時間を奪わない」）。**`prefers-reduced-motion` 対応必須（§13/§17）**。 |

## 3. グリッド＆スペーシング

> **実装状況** 【実装済】 — `--sp-*` / `--content-max`（段階拡大）/ `--gutter` トークン実装済み。セクション区切りは罫線でなく余白（§3.3）。

> **原則: 余白に意味を持たせる＝近接は関係性。** すべての余白はトークンとグリッドに従い、「なぜこの間隔か」を説明できること。

### 3.1 スペーシングスケール（4px 基底トークン）

| トークン | 値   | **用途（この間隔をどこで使うか）**                                  |
| -------- | ---- | ------------------------------------------------------------------- |
| `--sp-1` | 4px  | アイコンと数値の隙間・バッジ内パディング・メタの塊内（行ガター）    |
| `--sp-2` | 8px  | チップ/ボタン間隔・カード操作アイコンの余白・**強く関係する要素間** |
| `--sp-3` | 12px | カード内パディング・list-row の内側・メタの塊間（列ガター）         |
| `--sp-4` | 16px | セクション**見出しと中身**の間・要素グループ内の標準間隔            |
| `--sp-5` | 24px | セクション内ブロック間・カードレールのアイテム間                    |
| `--sp-6` | 32px | ブロック大間隔                                                      |
| `--sp-7` | 48px | **セクション間の縦リズム**（標準）。隣接セクションの境              |
| `--sp-8` | 64px | 最大余白（必要時のみ）                                              |

- **恣意的な px を直書きしない**。新しい間隔が要るならまずスケールで表現できないか検討する。
- **近接ルール**：関係が強い要素ほど小さい間隔（`--sp-1`〜`--sp-3`）、独立したセクション同士は大きい間隔（`--sp-7`）。
- **メタの塊（アイコン＋数値）**は塊内 `--sp-1`・塊間 `--sp-3` で並べ、塊の区切りを余白で見せる（`.card-caption`／`.meta`）。

### 3.2 コンテナ・ガター・整列

| トークン        | 値                                             | 用途                                                       |
| --------------- | ---------------------------------------------- | ---------------------------------------------------------- |
| `--content-max` | `1280px`（広幅で `1520→1800→2200` に段階拡大） | 全セクションの**中身の最大幅**。画面が広いほど横を活用する |
| `--gutter`      | `clamp(1rem, 5vw, 4rem)`                       | 画面左右の最小余白（流体）。全セクション共通               |

- **全セクションは同一の `--content-max` ＋ `--gutter` に揃える**。見出し・レール・リスト・チップ群・ボタン列の**左端・右端・カラム位置が画面全体で一致**すること。
- ワイド画面では中身を `max-width` で中央寄せし、左右に `--gutter`。**超広幅では `--content-max` を段階拡大**し、一覧グリッドの列数も増やして横を使い切る（§12）。
- **横スクロール帯（カードレール）も左端＝`--gutter` から開始**し、他セクションの左端と揃える（`scroll-padding-inline: var(--gutter)`）。

### 3.3 セクション間の縦リズム（罫線を使わない区切り）

- セクション間は **`--sp-7`（48px）を標準**とし、必要に応じてセクション内パディング `clamp(--sp-5, 4vw, --sp-7)`。
- **セクションの区切りは罫線（`border-bottom`）でなく余白で表す**（`section[data-section]:not(.hero)` の padding を拡大）。装飾の罫線は引かない（ヘッダ/フッタ等の機能的境界を除く）。
- **ヒーロー検索とクイックアクセスの間は近接させる**（両者は「探し始める」入口として関係が強い）。検索 → クイックアクセス → 看板（TOP10）の順に、関係の強さに応じて間隔を段階化する。

## 4. タイポグラフィ

> **実装状況** 【実装済】 — IBM Plex Sans JP 実装済み。4 段スケール `style.css` 適用済み。

### 4.1 フォント

**IBM Plex Sans JP**（Fontsource 自己ホスト・400/500/700）。`style.css` 先頭で `@import '@fontsource/ibm-plex-sans-jp/{400,500,700}.css'`。**Vite の `assetsInlineLimit:0`** で woff2 を `data:` にインライン化せず self ファイル化（CSP `font-src 'self'` 準拠）。unicode-range 分割で日本語グリフは必要時のみ DL。数値は `font-variant-numeric: tabular-nums`（再生数・順位・件数・話数の桁揃え）。**日本語に italic は使わない**。

### 4.2 4段スケール（隣接段の差を一目で）

| 段     | トークン    | 値                                 | **この段だけが担う役割**                                                                             |
| ------ | ----------- | ---------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **L1** | `--text-l1` | `clamp(1.75rem, 4vw, 2.5rem)`      | **他ページの「顔」用に温存**（トップはヒーロー見出しを置かない方針＝§10）                            |
| **L2** | `--text-l2` | `clamp(1.125rem, 2.5vw, 1.375rem)` | **セクション見出し `h2`**・詳細の作品名・モーダル見出し                                              |
| **L3** | `--text-l3` | `clamp(0.875rem, 1.5vw, 1rem)`     | **本文・カード/行タイトル・入力・主要ボタン文字**。可読の基準                                        |
| **L4** | `--text-l4` | `clamp(0.75rem, 1.2vw, 0.8125rem)` | **メタ**（再生数・話数・投稿時間・件数・タグ・小見出し・パンくず・すべて見る）※最小 12px・~11px 回避 |

> 隣接段で**最低 1.25 倍**の差を保ち、階層を視覚的に即読み取れるようにする。中間段を増やさない。
> L4 の下限は 12px（`0.75rem`）以上とし、**11px 以下になるサイズは使用しない**（メタの可読性確保）。

### 4.3 ウェイトルール（400/500/700）

| ウェイト                | 用途                                                                                                  | 禁止                       |
| ----------------------- | ----------------------------------------------------------------------------------------------------- | -------------------------- |
| **400 本文**            | あらすじ・各話/カードの**メタ数値**・入力テキスト                                                     | 見出しに使わない           |
| **500 ラベル/タイトル** | カード/行タイトル・ボタン文字・アクティブな並び替えラベル・ナビリンク・**各話行の投稿時間（強調用）** | —                          |
| **700 見出し/ランク**   | セクション `h2`・ロゴ・**ランク数字**・プライマリボタン・小見出しラベル・バッジ                       | 本文に使わない（うるさい） |

| `--shadow-sm` / `--shadow-md` | `0 1px 3px /0.4` / `0 4px 12px /0.5` | `…/0.1` / `…/0.12`         | 静止時の浮き / ホバー・モーダル                                                       |

**画像オーバーレイ専用の固定色**（テーマ非依存）：scrim＝`oklch(0 0 0/α)`、白文字＝`oklch(0.97〜1 0 0)`。画像上のコントラストはテーマで変えないため semantic を通さない。

### 5.3 アクセント・ポリシー（azure 1色）

- **使用限定**：リンク文字／アクティブ状態（タブ・ページ・並び替え選択・シリーズバッジ）／`:focus-visible`／プライマリボタン（詳細の「公式」「一覧で探す」）／ロゴのマーク／ヒーロー背景のごく淡い radial／メタアイコンの hover。
- **♥/✓ の ON は azure ではなく専用色**（♥＝`--heart` 赤塗り・✓＝`--watched` 緑塗り＝§9.7）。意味の異なる状態を1色に混ぜない。
- **禁止**：装飾的ベタ塗り・カラーブロック・通常ボーダー（→`--border`）。
- **コントラスト**：azure はダーク/ライト各背景で文字対比 ≥ 4.5:1（WCAG AA）。`--accent` 上の文字は `--bg`。

## 6. テーマ機構

> **実装状況** 【実装済】 — `data-theme` 3 状態・FOUC 防止・テーマトグル・View Transitions（reduced-motion ガード付き）全実装済み。

### 6.1 `data-theme` 3 状態

| 状態                 | 機構                            | 効果       |
| -------------------- | ------------------------------- | ---------- |
| 属性なし             | `@media (prefers-color-scheme)` | OS 追従    |
| `data-theme="dark"`  | `[data-theme="dark"]`           | 強制ダーク |
| `data-theme="light"` | `[data-theme="light"]`          | 強制ライト |

`<html>` に `color-scheme: dark light` を宣言し、ネイティブ UI（スクロールバー・フォーム）も追従。

### 6.2 FOUC 対策（`theme-init.js`）

`<head>` で**同期ロード**（`async`/`defer` 不可）。body 描画前に `localStorage('nico-danime-theme')` を読んで `data-theme` を確定。`script-src 'self'` 準拠の外部ファイル。

### 6.3 切替アニメーション

`theme.ts` の `toggleTheme()` が `document.startViewTransition`（あれば）で切替。**`prefersReducedMotion()`（`matchMedia('(prefers-reduced-motion: reduce)')`）が true なら View Transitions をスキップして即時切替**にする。CSS は `@supports (view-transition-name: none)` ガードで `::view-transition-old/new(root){ animation-duration:0.2s }`。未対応は即時切替（劣化なし）。

### 6.4 ヘッダのアイコン順

右上は左→右で **`🔍 → ☀/🌙（テーマ）→ ⚙（設定）`**。慣例どおり「頻用トグル＝内側、設定＝右端」。詳細動線は `screens.md`。

### 8.2 メタのアイコン圧縮（アイコン＋最小の単位語）

メタ情報は冗長なラベルを書かず、**アイコン＋最小の単位語**に圧縮する（`components/meta.ts` の `metaSpan()`）。
**純アイコンのみ（数値も単位語も無し）は不可** ── 視覚は圧縮するが支援技術には `aria-label` で言語補完する（§17）。

共通パターン `.meta`：

```
.meta            … inline-flex・gap var(--sp-1)・色 --text-muted・L4・tabular-nums
.meta svg        … 0.85em 角・currentColor（数値と同色・同サイズ感）
.meta.emphasize  … --text／weight 500（各話行の投稿時間など「主役のメタ」を1つだけ強調）
複数メタは gap var(--sp-3) で塊ごとに区切る（中黒なし）
```

| 意味     | 表現（アイコン＋最小単位語） | 例                                     | NG（冗長）             |
| -------- | ---------------------------- | -------------------------------------- | ---------------------- |
| 再生数   | `[play] 数値`                | `▶ 30万` / `▶ 9,876`                   | ~~「30万 再生」~~      |
| 話数     | `[film] 数値話`              | `▣ 12話`（「全」は省く・「話」は残す） | ~~「全12話」~~         |
| 投稿時間 | `[clock] 相対時間`           | `🕐 3時間前` / `🕐 6/16`               | ~~「6月16日投稿」~~    |
| コメント | `[message] 数値`             | `🗨 456`                               | ~~「456 コメント」~~   |
| マイリス | `[bookmark] 数値`            | `🔖 123`                               | ~~「123 マイリスト」~~ |

- 数値整形（`meta.ts`）：再生数/コメント/マイリスは概数（`formatViews`：1万以上は「N万」・未満は3桁区切り）、話数は実数、尺は `formatDuration`（「N分」/「N時間N分」）、投稿時間は相対（`formatRelativeTime`）。
- **アクセシビリティ**：アイコンは `aria-hidden`。意味は `.meta` 要素に `aria-label`（`MetaSpec.label`）で与える（例 `再生数 30万`）。視覚は圧縮、支援技術には言語で補う（§17）。

### 9.11 クイックアクセスのマーキー（`components/marquee.ts`）

> **実装状況** 【実装済】 — `overflow-x` のネイティブスクロール＋rAF 自動送り。CSS transform アニメは使わない。

- 2段目はランダムタグ（`discoveryTags` から `#`付きでサンプル）を**横スクロール帯**で出す。同じタグ並びを2回敷き、track 半分でラップしてシームレスループ。

- **一時停止**：hover（pointerenter）・操作中（pointerdown/touchstart）は自動送りを止め、離すと再開。
- **`prefers-reduced-motion: reduce`** では自動送りせず手動スクロールのみ。
- 再レンダリングで rAF を自動停止（リーク防止）。

### 17.4 Skip Link

- ページ先頭に Skip Link `<a href="#main-content" class="skip-link">` を置き、`:focus` で画面上部に出現（accent 背景）。`#main-content` は `<main>`。

### 17.6 `prefers-reduced-motion`

`@media (prefers-reduced-motion: reduce)` で View Transitions・カードのホバーズーム/浮き上がり・マーキー自動送りを無効化し、その他 transition を最短化する。テーマ切替は `prefersReducedMotion()` を確認して View Transitions をスキップ（§6.3）。

## 19. 一覧のフィルタ・並び替え（現状仕様）

> **実装状況** 【実装済】 — `list.ts` / `filter.ts` / `router.ts` / `main.ts`。

### 19.1 並び替え（`sort` ＋ `dir`）

- キー：`hot`（既定・ラベル「Hot」）／`views`（「累計再生数」）／`new`（「新着」）／`comments`（「コメント数」）／`kana`（「五十音順」）。**2 列レイアウト**で提示。
- **昇順/降順は共通トグル**（項目を倍にせず選択キーに作用）。**既定＝降順（desc）**。`asc` は降順結果の全反転。適用中バーに方向（昇順/降順）を表示。
- `new` は `latestAt`（無ければ `firstAt`）降順・タイブレーク seriesId。`comments` は `commentTotal` 降順。`kana` は `colKey`（行 char）順→日本語タイトルで `localeCompare`。

### 19.2 フィルタ

- **検索（タグ・トークン入力）**：`#` でタグ候補オートコンプリート（`seriesCount` 降順・最大 10）→ 確定タグはピル（`.tag-pill`・×で個別削除）。**複数タグは AND**（`every`）。プレーン文字は作品名検索（`q`）。サイドバーのタグ選択・適用中バーと同期。
- **タグ（サイドバー）**：出現数（`seriesCount`）降順・段階表示（§9.9）。クリックでトグル。
- **クール**：段階表示（§9.9）。クリックでトグル。`current`/`previous` キーワード対応。
- **五十音（`row`）**：行 char（あ/か/さ…＋全）で `colKey` 一致絞り。行内は厳密な50音順にできず**タイトル文字列順フォールバック**。
- **再生時間レンジ**：平均話長（`durationTotal/episodeCount` 分）の離散スナップ 2 ハンドル。停止点＝**下限なし/5分/15分/30分/1時間/2時間/上限なし(∞)**。
- **投稿年レンジ**：`firstAt`（無ければ `latestAt`）の年で離散スナップ 2 ハンドル。
- **お気に入りだけ / 未視聴だけ**：チェックで絞り（横並び）。
- **空シェル表示**（「中身のない項目も表示」）：既定 OFF＝`episodeCount > 0` のみ表示（サムネ/最新話/初出すべて欠落の殻 96 件を除外）。ON で全件（`includeEmpty`）。データは保持し表示側だけで切替。

### 19.3 表示件数・ページング・URL

- **表示件数**：既定 **50**・セレクタ **50/100/200**。
- ページング（無限スクロールにしない）。`paginateWorks`。
- **URL クエリで再現**：`q` / `row` / `tag`（カンマ区切り＝複数 AND）/ `cours` / `sort`（既定 hot は省略）/ `dir`（既定 desc は省略）/ `page`（既定 1 は省略）/ `size`（既定 50 は省略）。
- **インメモリ（URL 非再現）**：お気に入り/未視聴/空シェル表示/再生時間・投稿年レンジ（`fav=1` のみクイックアクセス起動用に URL から受ける）。

### 19.4 テキスト省略・折返し

- カード/list-row タイトル：`line-clamp: 2`。
- ピル：1 行・`min(20ch,100%)`・超過 `…`（全文はツールチップ＝§9.12）。
- あらすじ・各話あらすじ：`white-space: pre-line`（`<br>`→改行を保持）＋`word-break: auto-phrase`（§13）。

### 19.5 作品詳細（各話・あらすじ・関連）

> **実装状況** 【実装済】 — `detail.ts` / `main.ts` / `export.mjs`。

- **行レベルの総再生数は出さない**（各話ドロワーに集約）。シリーズメタは [film]話数・[message]総コメント・[bookmark]総マイリス（**総再生数は出さない**）。
- **各話ドロワー（アコーディオン）**：本体クリックで詳細をインライン展開（chevron 90°）。中身＝サムネ（▶）・[play]再生数・[message]コメント・[bookmark]マイリス・[clock]投稿時間・[film]尺・**各話あらすじ**（`episodes[].description`）。各話の外部ボタンは **↗（external-link）**＝公式 watch。
- **あらすじ**：シリーズの第1話あらすじを**メタの下の独立ブロック**（`.detail-synopsis`／「あらすじ」ラベル＋本文）。**デスクトップ（≥768px）は常時展開・モバイルは折りたたみ**（`<details>` を CSS メディアクエリだけで切替し JS は open を触らない。UA の `details:not([open])` 非表示に詳細度で勝つよう `:not([open])` を含めて上書き）。改行保持＋`word-break:auto-phrase`。
