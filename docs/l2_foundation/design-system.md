## 2. ビジュアルポリシー（カラー基盤）

### 2.1 ダークテーマ

- **背景**: OKLCH 輝度 0.16、彩度 0.005、色相 260 のチャコール（ほぼ無彩・微青み）
- **カード面**: 輝度 0.21（背景より +0.05 の差で奥行き感を出す）
- **テキスト**: 輝度 0.90（WCAG AA 以上をダーク面上で確保）
- **ボーダー**: 輝度 0.30（微弱な区切り線として機能）

### 2.2 ライトテーマ

- **背景**: OKLCH 輝度 0.97、彩度 0.002、色相 260 のオフホワイト（純白でなく微青みグレー）
- **カード面**: 輝度 1.00（純白。背景より明るくして浮かせる）
- **テキスト**: 輝度 0.18（濃いニュートラル。WCAG AA 以上）
- **ボーダー**: 輝度 0.88（薄い区切り線）

- 色: azure blue（ダーク `oklch(0.65 0.13 240)` / ライト `oklch(0.48 0.16 245)`）
- WCAG: ダーク背景上で対比率 ≥ 4.5:1、ライト背景上で対比率 ≥ 4.5:1（AA 準拠）

- **使用禁止**: 装飾的な背景・カラーブロック・ボーダー全般（`--border` を使う）

## 4. テーマ機構

### 4.1 `data-theme` 3 状態

| 状態                  | 機構                                     | 効果        |
| --------------------- | ---------------------------------------- | ----------- |
| `data-theme` 属性なし | `@media (prefers-color-scheme)` に委ねる | OS 設定追従 |
| `data-theme="dark"`   | `[data-theme="dark"]` セレクタ           | 強制ダーク  |
| `data-theme="light"`  | `[data-theme="light"]` セレクタ          | 強制ライト  |

CSS 構造:

```css
/* デフォルト: ダーク（未設定時は OS 設定追従でダーク優先） */
:root { --bg: var(--p-ch-950); ... }

/* OS がライトを要求 かつ 明示指定なし → ライトトークンに上書き */
@media (prefers-color-scheme: light) {
  :root:not([data-theme="dark"]) { --bg: var(--p-ln-50); ... }
}

/* 明示的な上書き（OS 設定無視） */
[data-theme="dark"]  { --bg: var(--p-ch-950); ... }
[data-theme="light"] { --bg: var(--p-ln-50); ... }
```

### 4.2 FOUC 対策 (`web/public/theme-init.js`)

```javascript
// theme-init.js — <head> で同期ロード（async/defer 不可）
// script-src 'self' 準拠
;(function () {
  var v = localStorage.getItem('nico-danime-theme')
  if (v === 'dark' || v === 'light') {
    document.documentElement.dataset.theme = v
  }
})()
```

- `<head>` の `<script src="/theme-init.js">` で同期実行 → body レンダリング前に `data-theme` が確定
- `localStorage` 未設定時は属性なし → OS 設定追従

### 4.3 テーマ切り替えアニメーション

`theme.ts` の `applyTheme()` で `document.startViewTransition` を使用:

```typescript
if (typeof document.startViewTransition === 'function') {
  document.startViewTransition(() => {
    root.dataset.theme = theme
  })
} else {
  root.dataset.theme = theme
}
```

CSS: `::view-transition-old/new(root) { animation-duration: 0.2s }` (@supports ガード)

## 5. タイポグラフィ

### 5.1 フォント: IBM Plex Sans JP (Fontsource 自己ホスト)

```
pnpm add @fontsource/ibm-plex-sans-jp
```

style.css の先頭:

```css
@import '@fontsource/ibm-plex-sans-jp/400.css';
@import '@fontsource/ibm-plex-sans-jp/500.css';
@import '@fontsource/ibm-plex-sans-jp/700.css';
```

Fontsource が unicode-range 分割 `@font-face` を生成するため、日本語グリフは日本語テキストが出現したときのみダウンロードされる。ビルド時に Vite が `dist/assets/` にフォントファイルをコピー。

### 5.2 タイポスケール（4段）

| レベル | トークン    | 値                                 | 役割                               |
| ------ | ----------- | ---------------------------------- | ---------------------------------- |
| L1     | `--text-l1` | `clamp(1.75rem, 4vw, 2.5rem)`      | ヒーロー見出し（ページに1箇所）    |
| L2     | `--text-l2` | `clamp(1.125rem, 2.5vw, 1.375rem)` | セクション見出し（h2）             |
| L3     | `--text-l3` | `clamp(0.875rem, 1.5vw, 1rem)`     | 本文・カードタイトル・入力フォント |
| L4     | `--text-l4` | `clamp(0.6875rem, 1.2vw, 0.75rem)` | メタ情報・タグ・ページング         |

### 5.3 ウェイトルール

| ウェイト | 使用箇所                                                         |
| -------- | ---------------------------------------------------------------- |
| 400      | 本文・カード本文・インプット内テキスト                           |
| 500      | ラベル・アクティブタブ・カードタイトル強調・ナビゲーションリンク |
| 700      | セクション h2・ヒーロー h1・ロゴ                                 |

**日本語テキストには italic を使わない**（IBM Plex Sans JP はイタリック非対応）。

## 8. モダン CSS ポリシー

採用する機能と理由:

| 機能             | 使用箇所                           | 採用理由                                                    |
| ---------------- | ---------------------------------- | ----------------------------------------------------------- |
| `@layer`         | style.css 全体                     | reset→tokens→base→components→utilities の優先順位を明示制御 |
| CSS nesting      | コンポーネントルール内             | セレクタの関係性を局所化・バンドルサイズ削減                |
| `@container`     | `.series-card` 内                  | カード幅でタイトル表示行数を変化させる                      |
| `:has()`         | フィルタ状態に応じたレイアウト変化 | 状態クラスを親から検知                                      |
| `aspect-ratio`   | サムネイル・バナー                 | レイアウトシフト防止                                        |
| `scroll-snap`    | カードレール                       | 横スクロール時の停止位置固定                                |
| `clamp()`        | タイポスケール                     | メディアクエリなしで流体的スケール                          |
| `oklch()`        | カラートークン primitive 層        | 知覚的均等色空間・ライト計算                                |
| View Transitions | テーマ切り替え                     | クロスフェードを1行で記述（`@supports` ガード付き）         |
| `dvh` / `dvw`    | モーダル高さ                       | モバイル動的ビューポートに対応                              |

禁止:

- `unsafe-inline` に依存するインラインスタイル（`style=""` 直接記述）
- `!important`（レイヤー順で解決する）
- CSS Hacks（vendor prefix 非標準形）

## 9. アイコン（Lucide 互換バンドル SVG）

**実装**: `web/src/components/icon.ts` に Lucide のSVGパスをハードコード。

- runtime fetch なし（`img-src` 拡張不要）
- CSP `style-src 'self'` / `script-src 'self'` 準拠
- 各アイコンは `<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">` ベース

| 名前            | 用途                           |
| --------------- | ------------------------------ |
| `search`        | ヘッダ検索ボタン・ヒーロー入力 |
| `settings`      | ヘッダ設定ボタン               |
| `sun`           | テーマボタン（ライト表示中）   |
| `moon`          | テーマボタン（ダーク表示中）   |
| `heart`         | カードお気に入りボタン         |
| `check`         | カード見たボタン               |
| `external-link` | 公式リンクボタン               |
| `shuffle`       | タグシャッフルボタン           |
| `info`          | 補足情報ボタン                 |
| `x`             | モーダル閉じるボタン           |
| `chevron-right` | パンくず・「すべて見る」リンク |

API: `icon(name: IconName, size?: number): SVGElement`
内部 SVG 文字列生成: `iconSvg(name: IconName, size?: number): string`（テンプレートリテラル内使用）

ライセンス: Lucide は ISC ライセンス。SVGパスの使用・変形・頒布が許可されている。
