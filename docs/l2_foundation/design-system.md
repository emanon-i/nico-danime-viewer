## 0. 設計原則（このシステムが守ること）

| 原則                      | 具体                                                                                                                                                        |
| ------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **画像が主役・UI は黒子** | 色はサムネに任せる＝UI は無彩（チャコール/オフホワイト）＋寒色アクセント1色のみ（方針A）。カードは**サムネ全面**で、文字は最小限を画像の下端にだけ重ねる。  |
| **定義より用途**          | トークンは「何に・どの状態で使うか」までを必ず併記。曖昧なトークンは置かない（§3・§5）。                                                                    |
| **DRY＝単一ソース**       | カード/チップ/アイコンボタン/セクション見出し/レールは Factory（`components/`）＋共通 CSS クラス1系統。インラインスタイル禁止（CSP）。                      |
| **予測どおり**            | ホバーで隠れた要素を出すのは可（操作アイコン）だが、タッチ端末では常時表示にフォールバック。レイアウトシフトを出さない（`aspect-ratio`＋ `width/height`）。 |

| 項目           | 内容                                                                                              |
| -------------- | ------------------------------------------------------------------------------------------------- |
| 対象リポジトリ | `web/src/` 配下の静的フロントエンド（vanilla TS・独自 router・Vite）                              |

| FOUC 防止      | `web/public/theme-init.js`（`<head>` 同期ロード）                                                 |
| 検証スクリプト | `scripts/shoot-top.mjs`（Playwright・画像ロード完了待ち→4枚撮影）                                 |
| 対象ブラウザ   | Chrome 112+ / Firefox 117+ / Safari 16.5+（CSS nesting・OKLCH・`color-mix`・`:has()` ネイティブ） |

### 1.1 CSP 制約（厳格・維持）

`web/index.html` の `<meta http-equiv="Content-Security-Policy">`：

```
default-src 'self'; script-src 'self'; style-src 'self'; connect-src 'self';
img-src 'self' https://nicovideo.cdn.nimg.jp https://*.nimg.jp;
base-uri 'self'; object-src 'none'
```

- `style-src 'self'` → **インライン `style=""` 禁止**。色・寸法はすべて `style.css` のクラス/トークンで表現。
- `script-src 'self'` → **インライン `<script>` 禁止**。`theme-init.js` も外部ファイルを同期ロード。
- `img-src` → サムネ実ホスト（§2）を許可済み。**追加変更は不要**（診断で確認）。

## 2. サムネイル診断（実データ実測・最重要）

`data/works.json`（6,352 シリーズ）と `data/ranking.json` を実測した結果。

### 2.1 実ホスト

| 項目              | 実測値                                                                                      |
| ----------------- | ------------------------------------------------------------------------------------------- |
| サムネ URL ホスト | **`nicovideo.cdn.nimg.jp`（非 null 6,256 件の 100%）**                                      |
| サムネ欠損        | 96 件（`thumbnailUrl: null`）→ 無地フォールバック（§6.3）                                   |
| CSP `img-src`     | `https://nicovideo.cdn.nimg.jp` と `https://*.nimg.jp` を**既に許可済み＝正しい**。追加不要 |

→ **旧プレビューでサムネが出なかった原因は CSP でもホストでもデータでもない。** URL は HTTP 200 / `image/jpeg` を返し、CSP も通る。原因は**スクリーンショット側が画像ロードを待たずに撮影していた**こと（＋`loading="lazy"` で初期ビューポート外は未ロード）。本システムは検証スクリプトで**画像の `complete && naturalWidth>0` を待ってから撮影**して再発を防ぐ（§9）。

### 2.2 実アスペクト比とサイズ・サフィックス

ニコニコ CDN のサムネは URL サフィックスで解像度が変わる。実測：

| URL 形                           | 実寸         | 比率             | 用途                            |
| -------------------------------- | ------------ | ---------------- | ------------------------------- |
| 素 `…/<id>.<rev>`（data 内の値） | **130×100**  | 1.30             | 小さく粗い → カード全面には不適 |
| `…/<id>.<rev>.M`                 | 320×240 相当 | 4:3              | 中                              |
| **`…/<id>.<rev>.L`**             | **360×270**  | **4:3（1.333）** | **カード全面に採用**            |

- **画像ファイルの実寸は 4:3（360×270）だが、中身は 16:9 映像を上下黒帯でレターボックスした焼き込み**（実画像を取得して確認＝ぼっち/SPY/HxH すべて上下に黒帯。原因はソース側＝こちらの CSS ではない）。**よってカードは 16:9 を採用し、`object-fit:cover` で上下黒帯（各 1/8＝33.75px）を正確にトリミング**して実コンテンツだけを全面表示する（§6.4）。
- **`.L` 昇格を必須化**：`components/card.ts` の `hiResThumb()` が素 URL（`/thumbnails/\d+/\d+\.\d+$`）を検出して `.L` を付ける。既にサフィックス付き/別ホストはそのまま。`.L` は同一ホストなので CSP 追加不要。

```ts
export function hiResThumb(url: string | null): string {
  if (!url) return ''
  return /\/thumbnails\/\d+\/\d+\.\d+$/.test(url) ? `${url}.L` : url
}
```

## 3. カラートークン（OKLCH・3層）

色は **primitive（パレット定数）→ semantic（用途トークン）→ component（参照）** の3層。コンポーネントは semantic のみ参照する。

| ------------------------------------------------------ | ------------------------------------------------------- | ----------------------------------------------- |

| `--p-ch-900`                                           | `oklch(0.21 0.005 260)`                                 | チャコール カード/パネル面                      |
| `--p-ch-850`                                           | `oklch(0.25 0.006 260)`                                 | チャコール ホバー/画像欠損面                    |
| `--p-ch-700`                                           | `oklch(0.30 0.005 260)`                                 | チャコール ボーダー                             |
| `--p-ch-400`                                           | `oklch(0.60 0.005 260)`                                 | チャコール ミュートテキスト                     |
| `--p-ch-200`                                           | `oklch(0.82 0.005 260)`                                 | チャコール セカンダリテキスト                   |
| `--p-ch-100`                                           | `oklch(0.90 0.005 260)`                                 | チャコール プライマリテキスト                   |

### 3.2 セマンティック層（用途・状態を明記）

| トークン         | ダーク                        | ライト                     | **使用箇所（何に・どの状態で）**                                           |
| ---------------- | ----------------------------- | -------------------------- | -------------------------------------------------------------------------- |

| `--surface`      | `--p-ch-900`                  | `--p-ln-0`                 | 入力欄・チップ・ボタン・パネルの**面**。カード自体は画像が面なので使わない |
| `--surface-2`    | `--p-ch-850`                  | `--p-ln-100`               | ホバー面・**サムネ欠損のプレースホルダ面**・アイコンボタンホバー           |
| `--text`         | `--p-ch-100`                  | `--p-sl-900`               | プライマリ文字（見出し・本文・ボタン文字）                                 |
| `--text-muted`   | `--p-ch-400`                  | `--p-sl-500`               | 副次文字・プレースホルダ・セクション小見出し・ⓘ・非アクティブアイコン      |
| `--border`       | `--p-ch-700`                  | `--p-ln-200`               | 区切り線・入力/チップ/ボタンの輪郭。**装飾の色付けには使わない**           |
| `--accent`       | `--p-az-dk`                   | `--p-az-lt`                | §3.3 の限定用途のみ                                                        |
| `--accent-hover` | `--p-az-dk-h`                 | `--p-az-lt-h`              | アクセント要素の `:hover`                                                  |
| `--focus-ring`   | `oklch(0.65 0.13 240/0.5)`    | `oklch(0.48 0.16 245/0.4)` | `:focus-visible` アウトライン・入力フォーカスの glow                       |
| `--shadow-sm`    | `0 1px 3px oklch(0 0 0/0.4)`  | `…/0.1`                    | カード/入力 静止時の浮き                                                   |
| `--shadow-md`    | `0 4px 12px oklch(0 0 0/0.5)` | `…/0.12`                   | カードホバー・モーダル・ヘッダ検索                                         |

**画像オーバーレイ専用の固定色**（テーマ非依存・画像の上は常に暗背景＋白文字）：
`oklch(0 0 0 / α)` の scrim と `oklch(0.97〜1 0 0)` の白文字を使う。これらは semantic トークンを通さない（画像上のコントラストはテーマで変えてはいけないため）。

### 3.3 アクセント・ポリシー（azure 寒色1色）

- **使用限定**：リンク文字 / アクティブ状態（タブ・ページ・♥✓ ON）/ `:focus-visible` / プライマリボタン（詳細の「公式」）/ ロゴのマーク / ヒーロー背景のごく淡い radial（14% 透過）。
- **使用禁止**：装飾的な背景ベタ塗り・カラーブロック・通常のボーダー（→ `--border`）。
- **コントラスト**：azure はダーク/ライト各背景で文字対比 ≥ 4.5:1（WCAG AA）。`--accent` 上の文字は `--bg`（プライマリボタン）。

## 4. テーマ機構

### 4.1 `data-theme` 3 状態

| 状態                 | 機構                            | 効果       |
| -------------------- | ------------------------------- | ---------- |
| 属性なし             | `@media (prefers-color-scheme)` | OS 追従    |
| `data-theme="dark"`  | `[data-theme="dark"]`           | 強制ダーク |
| `data-theme="light"` | `[data-theme="light"]`          | 強制ライト |

加えて `<html>` に **`color-scheme: dark light`** を宣言し、ネイティブ UI（スクロールバー・フォーム）も追従させる。

### 4.2 FOUC 対策（`theme-init.js`）

`<head>` で**同期ロード**（`async`/`defer` 不可）。body 描画前に `localStorage('nico-danime-theme')` を読んで `data-theme` を確定。`script-src 'self'` 準拠の外部ファイル。

### 4.3 テーマ切替アニメーション

`theme.ts` の `applyTheme()` が `document.startViewTransition`（あれば）で `data-theme` を切替。CSS は `@supports (view-transition-name: none)` ガードで `::view-transition-old/new(root) { animation-duration: 0.2s }`。未対応ブラウザは即時切替（劣化なし）。

## 5. タイポグラフィ

### 5.1 フォント

**IBM Plex Sans JP**（Fontsource 自己ホスト・400/500/700）。`style.css` 先頭で `@import '@fontsource/ibm-plex-sans-jp/{400,500,700}.css'`。unicode-range 分割で日本語グリフは必要時のみ DL。数値は `font-variant-numeric: tabular-nums`（再生数・順位・件数の桁揃え）。**日本語に italic は使わない**（非対応）。

### 5.2 4段タイポスケール（差が一目で分かる）

| 段     | トークン    | 値                                 | **用途（この段だけが担う役割）**                                                                     |
| ------ | ----------- | ---------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **L1** | `--text-l1` | `clamp(1.75rem, 4vw, 2.5rem)`      | **ヒーロー見出しのみ**（1ページ1箇所）。ページの「顔」。                                             |
| **L2** | `--text-l2` | `clamp(1.125rem, 2.5vw, 1.375rem)` | **セクション見出し `h2`**（人気TOP10 / 最近追加 / クール / タグ）・詳細の作品名・モーダル見出し。    |
| **L3** | `--text-l3` | `clamp(0.875rem, 1.5vw, 1rem)`     | **本文・カードタイトル・入力フォント・クール/プライマリボタン文字・各話タイトル**。可読の基準。      |
| **L4** | `--text-l4` | `clamp(0.6875rem, 1.2vw, 0.75rem)` | **メタ情報**（再生数・件数・タグチップ・セクション小見出し・パンくず・ページング・「すべて見る」）。 |

> L1↔L2 は約2倍、L3↔L4 は約 0.75 倍。隣接段で**最低でも 1.25 倍の差**を保ち、階層が視覚的に即読み取れる。中間段を増やさない（曖昧化を防ぐ）。

### 5.3 ウェイトルール（400/500/700 の使い分け）

| ウェイト | 使用箇所                                                                                   | 禁止                       |
| -------- | ------------------------------------------------------------------------------------------ | -------------------------- |
| **400**  | 本文・カードのメタ（再生数）・入力テキスト・各話タイトル・あらすじ                         | 見出しに使わない           |
| **500**  | カードタイトル・クイック/クール/タグ等のボタン文字・アクティブな並び替えラベル・ナビリンク | —                          |
| **700**  | ヒーロー h1・セクション h2・ロゴ・ランク数字・プライマリボタン・小見出しラベル             | 本文に使わない（うるさい） |

## 6. コンポーネント仕様（DRY・状態つき）

### 6.0 形状・余白・影・レイヤー

- **スペーシング**（4px 基底）：`--sp-1`=4 … `--sp-8`=64px。用途は `--sp-2`=チップ間隔、`--sp-3`=カード内パディング、`--sp-4`=セクション内要素間、`--sp-5`〜`--sp-7`=セクション余白。
- **コンテンツ幅**：`--content-max: 1280px` / `--gutter: clamp(1rem, 5vw, 4rem)`。セクションの**中身（見出し・レール・リスト・チップ群）を `max-width` で中央寄せ**し、ワイド画面でも間延び（スカスカ）させない。`--gutter` で左右の最小余白を流体化。
- **角丸**：`--r-sm`=4（チップ/バッジ）/ `--r-md`=8（カード・入力・ボタン）/ `--r-lg`=12（モーダル）/ `--r-full`（ピル・アイコンボタン・タグ）。
- **影**：`--shadow-sm`（静止）/ `--shadow-md`（ホバー・モーダル）。
- **z-index**：`--z-card-overlay`=1（カード上の操作）/ `--z-header`=10（sticky）/ `--z-modal`=100（モーダル・検索）。

### 6.1 シリーズカード `card()`（画像主役・本システムの中核）

**Factory**：`card(seriesId, title, thumbnailUrl, officialHref?, meta?)`、`meta?: { rank?: number; views?: number }`。
**DOM 契約**（テストが固定・変更不可）：

```
.series-card[data-series-id]            ← container-type: inline-size, aspect-ratio 4/3, overflow hidden
  a.card-body[href="?series=<id>"]      ← 主アクション＝うちの詳細。inset:0 で全面
    img.card-img                        ← .L サムネ・object-fit:cover・width/height=360/270
    .card-overlay                       ← 下端 scrim（文字の裏だけ暗く）。pointer-events:none
      span.card-rank?                   ← meta.rank 時のみ。大きな順位数字（cqw 連動）
      .card-text
        .card-title                     ← 2行 line-clamp・weight 500
        .card-meta?                     ← meta.views 時のみ「N 再生」（万表記）
  .card-actions                         ← 左上オーバーレイ
    button.card-favorite / button.card-watched
  a.card-external[target=_blank rel=noopener]  ← 右上オーバーレイ（副アクション＝公式 /series）
```

**視覚仕様**：

| 項目         | 仕様                                                                                                                                                                                    |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 比率         | **`aspect-ratio: var(--card-aspect)` ＝ 16/9**（素サムネ 4:3 の焼き込み黒帯を `cover` で除去。§6.4）                                                                                    |
| 画像         | `object-fit: cover`・ホバーで `scale(1.04)`（0.4s）。欠損時は `.no-thumb` で無地＋「NO IMAGE」                                                                                          |
| scrim        | `linear-gradient(to top, 0.85 → 0.6@38% → 0)`。**画像上部は暗くしない**。`pointer-events:none` でカードクリックを妨げない                                                               |
| ランク       | `clamp(1.5rem, 9cqw, 2.25rem)` weight700 tabular。`container-type: inline-size` によりカード幅 (`cqw`) に追従                                                                           |
| 操作アイコン | 32×32・`oklch(0 0 0/0.42)` ＋ `backdrop-filter: blur(4px)`。**既定は透明、`:hover`/`:focus-within` で表示**。`@media (hover: none)` では常時表示。`.active` は accent で fill＋常時表示 |
| 当たり判定   | ♥✓＝左上 / ↗＝右上 を**別位置**に置き重ねない（誤タップ防止）                                                                                                                           |

**状態**：

| 状態                                             | 視覚                                                                           |
| ------------------------------------------------ | ------------------------------------------------------------------------------ |
| `:hover` / `:focus-within`                       | カード `translateY(-3px)`＋`--shadow-md`、画像 `scale(1.04)`、操作アイコン出現 |
| `.card-favorite.active` / `.card-watched.active` | アイコン accent fill・常時表示（`localStorage` 同期は `main.ts` `wireCards`）  |
| `.no-thumb`                                      | 画像非表示・無地面＋ラベル                                                     |

**配置コンテキスト**：

- **レール内**（TOP10 等）：`.card-rail .series-card { flex: 0 0 clamp(180px, 24vw, 232px); scroll-snap-align: start }`。
- **グリッド内**（一覧）：`.list-grid .series-card { width: 100% }`（列数はブレークポイントで増加。§7）。

### 6.2 カードレール `.card-rail`（有限ランキング帯の横スクロール）

`display:flex; overflow-x:auto; scroll-snap-type:x proximity`。**スクロールバー非表示**（`scrollbar-width:none` ＋ `::-webkit-scrollbar{display:none}`）。両端を `mask-image` の linear-gradient でフェードし「続きがある」ことを示す。`scroll-padding-inline: var(--gutter)`。

### 6.3 サムネ欠損フォールバック

`thumbnailUrl` が null（96件）→ `card()` が `.no-thumb` を付与。`img` を `visibility:hidden` にし、`.card-body::before` に「NO IMAGE」を `--surface-2` 面で表示（`alt` 文字の散らかりを出さない）。詳細バナーは `.no-thumbnail`（4:3 の無地面）。

### 6.4 カード比率の決定（16:9・黒帯トリミング）

- `screens.md` は当初「カードは正方形(1:1)で統一」と記述（→ 16:9 に統一済み）。
- **当初 4:3 を採用していたが誤りだった**。理由は「実サムネが 4:3 なのでトリミングなしで全フレームを見せる」だったが、**前提が誤っていた**：素サムネ（`.L` 360×270）は **4:3 の枠に 16:9 映像を上下黒帯でレターボックスした焼き込み**で、「全フレーム表示」が実際には**黒帯を表示**していた（実画像を取得して確認＝§2.2）。
- **現在は 16:9 を採用**。`aspect-ratio: var(--card-aspect)`（= `16 / 9`）＋ `object-fit:cover` で、4:3 画像の上下（＝黒帯部分・各 1/8＝33.75px）を**正確にトリミング**し、16:9 の実コンテンツだけを全面表示する（数学的に黒帯ぴったり＝コンテンツ欠けなし）。
- **影響範囲**：トップ TOP10／一覧グリッド／関連シリーズ／`list-row` サムネ。比率変更は **`--card-aspect` 1トークン**で全箇所追従（実装は `.series-card`／`.list-row-thumb` が `var(--card-aspect)` を参照）。
- 留意：旧作などで素材が真に 4:3（黒帯なし）のものは上下がわずかに切れるが、現代アニメ（大多数）の 16:9 レターボックスを正しく扱うことを優先（標準的なサムネ挙動）。

## 7. ブレークポイント・レスポンシブ

| 名前 | px   | 切替                                                 |
| ---- | ---- | ---------------------------------------------------- |
| sm   | 480  | 一覧グリッド 2→3 列 / モバイル左サイドバー→ドロワー  |

- TOP10 レールはモバイル/PC/UW とも横スクロール（バー非表示）。一覧グリッドは縦グリッド。

## 8. モダン CSS ポリシー

| 機能                                            | 使用箇所                                           | 理由                                               |
| ----------------------------------------------- | -------------------------------------------------- | -------------------------------------------------- |
| `@layer reset,tokens,base,components,utilities` | 全体                                               | 優先順位を宣言順で制御（`!important` 不要）        |
| CSS nesting                                     | コンポーネント内                                   | 関係の局所化                                       |
| `@container` (`cqw`)                            | カードのランク数字                                 | カード幅に応じた可変タイポ                         |
| `color-mix(in oklch …)`                         | ヘッダ半透明・ボーダー減衰・recent 区切り          | トークンから派生色を生成（新トークンを増やさない） |
| `:has()`                                        | 一覧の並び替えラベル（`label:has(input:checked)`） | 親から状態検知                                     |
| `aspect-ratio`                                  | カード・バナー                                     | レイアウトシフト防止                               |
| `scroll-snap` / `mask-image` フェード           | カードレール                                       | 停止位置固定＋「続きあり」表現                     |
| `clamp()`                                       | タイポ・余白・カード幅・ヒーロー余白               | メディアクエリなし流体化                           |
| `oklch()`                                       | カラー primitive                                   | 知覚均等・明度設計                                 |
| `backdrop-filter`                               | ヘッダ・カード操作アイコン                         | 画像/背景の上での視認確保                          |
| `@media (hover: none)`                          | カード操作アイコン                                 | タッチ端末で常時表示にフォールバック               |
| View Transitions                                | テーマ切替                                         | 1行クロスフェード（`@supports` ガード）            |
| `dvh`                                           | モーダル高さ                                       | モバイル動的ビューポート                           |

**禁止**：インライン `style=""`（CSP）・`!important`・非標準 hack。
