# 画面設計（情報設計レベル）

> L2（`foundation.md`）から分離した**画面（ページ）構成**。各ページに **①ワイヤー（配置＋大きさ・モバイル/デスクトップ/ウルトラワイド）**
> と **②動線** の 2 層を持たせる。細かい CSS・コンポーネント実装は L3 以降。
> **掲載項目はすべてデータ可用性監査（`validation/api-availability.md`）で裏取り済みのものだけ**。
> **`ⓘ`= 出所ツールチップ**（各ラベル横の「?」。マウスオーバーでフィールドの取得元を表示。文言は本書末尾の出所マップ）。

## 画面一覧（v1＝3画面）

| 画面 | 役割 | 対応 REQ |
|------|------|---------|
| **1. トップ** | 入口・発見。検索＋クイックアクセス＋TOP10＋最近追加＋クール/タグ入口から一覧へ送る。テーマ切替 | REQ-0001/0002/0004/0008/0010/0012 |
| **2. 一覧** | 探索の主戦場。常設検索＋五十音＋左フィルタ（タグ/クール）＋右グリッド | REQ-0001/0002/0004/0008/0010/0011 |
| **3. 作品(シリーズ)詳細** | 各話一覧＋公式 deep-link＋同じタグへ回遊 | REQ-0001/0005（＋回遊で 0008） |

いずれも実行時は**事前生成した用途別 JSON を読むだけ**（API 直叩きなし）。**全状態は URL クエリで再現**（共有・ブックマーク・戻り位置が予測どおり）。
**ジャンル別フィルタは廃止**（監査: `genre` が 99.95%「アニメ」一色でサブジャンル判別不能 → タグ主軸）。

### クイックアクセスのプリセット・デフォルト

各ボタンは**一覧の特定プリセット状態（URL クエリ）**を開く。

| 入口 | フィルタ | 並び替え(デフォルト) | 備考 |
|------|---------|---------------------|------|
| **今期** | クール=現行季（`startTime` から自動判定） | 勢い順 | 今季の話題順 |
| **新着** | なし | 新着シリーズ順（シリーズ初出降順） | タイトル単位の新規 |
| **Hot** | なし | 勢い順（蓄積なし近似） | 「今の伸び」目安 |
| **人気TOP** | なし | 累計再生数順 | **全期間のみ＝期間セレクタ無し**（履歴を貯めないので月間/週間は作れない。「今」は Hot が担当） |
| 一覧 直アクセス | なし | 人気(累計)順 | 予測どおり重視（安定） |

### レスポンシブ注記

- **ヘッダの常設検索とテーマ切替（☀/🌙）は全幅で表示**。テーマ既定は `prefers-color-scheme`、選択は `localStorage` 保存（即時反映・遷移なし）。
- **PC/ウルトラワイド: 横スクロール廃止**。TOP10 ショーケースは「幅に収まる枚数を 1 行ベタ置き＋[すべて見る →]」。ウルトラワイドは⑤⑥⑦を 2 段組みに。
- **モバイルのみ** TOP10 を**横スワイプ**。クイックアクセスは 2×2 折返し、タグは折返し、クールは横スクロール。
- カードは**正方形(1:1)**で統一。

### 設計根拠（なぜこの順・この既定か）

- **②ヒーロー検索を最上部**: カタログが広く目的来訪が多い → 検索ファースト。コピーを「作品/コンテンツ」にしたのは支店が舞台/ライブ/旧作も含むため。
- **③今期を単独先頭にせず横並びボタンで両立**: 「今期/新着/Hot/人気TOP」は**対等な入口**。一つを固定先頭にすると他の動機が埋もれる → ボタン列で並置。
- **④人気TOP10ショーケース**: まず看板（定番の安心）を見せて回遊を喚起。**累計再生数順**＝予測どおりで安定。
- **⑤最近追加・更新**: 「変化」を提示して再訪動機を作る（新着シリーズ＝新規タイトル、最新話＝既存作の更新）。
- **⑥クール→⑦タグの順**: 季節は粒度が粗く入りやすい上位、タグはより能動的・細粒度なので下。**タグに舞台/ライブ等の非アニメ**を含め、アニメ以外の発見を掬う。
- **Hot と 人気TOP の役割分離**: Hot=「**今の勢い（蓄積なし近似）**」、人気TOP=「**全期間の定番（累計）**」。期間デルタを持てない制約を、2 ボタンの役割分担で正直に表現。
- **人気TOP に期間セレクタを置かない理由**: 履歴を蓄積しない方針 → 月間/週間ランキングは作れない。「今」の軸は Hot が担うので、TOP は全期間累計に限定（できない期間を UI に出さない＝予測どおり）。
- **ダーク/ライト**: 既定は OS 追従（`prefers-color-scheme`）、選択は `localStorage`。**テーマ設定のみで個人化データではない**（視聴履歴等は扱わない方針と矛盾しない）。

### ② 動線

- **入口 → トップ**。最初に目に入るのはヒーロー検索（検索ファースト）。
- **ヒーロー/ヘッダ検索入力 → 一覧**へ（`?q=…`）。
- **クイックアクセス [今期][新着][Hot][人気TOP] → 一覧の各プリセット状態**（上表の通り。URL クエリ。例 今期=`?cours=現行季&sort=hot`、人気TOP=`?sort=views`）。
- **TOP10／最近追加のカード → 作品詳細**（主）／カード隅 **[↗] → 公式シリーズ**（副・外部）。
- **クールボタン → 一覧のクール絞り**（`?cours=2026春`）／**タグボタン → 一覧のタグ絞り**（`?tag=舞台`）。
- **テーマトグル（☀/🌙）→ 即時反映＋`localStorage` 保存**（遷移なし）。
- 状態は **URL クエリで再現**（テーマのみ `localStorage`）。

### カード内訳（主/副アクション・全幅共通）

正方形カードは**主アクション＝カード本体、副アクション＝右上隅の小さな [↗] オーバーレイ**を併存させる。

```
 ┌──────────────┐
 │ thumb     [↗]│ ← 副: 右上隅オーバーレイ小ボタン [↗]
 │ (1:1)        │      → ニコニコ公式シリーズページ /series/<id> に直行（外部・詳細を経由しない）
 │              │      ↗ は外部リンクの明示
 │ 作品タイトル   │
 │ 勢い/再生 ⓘ   │ ← ⓘ 出所ツールチップ（[↗] とは別位置で併存）
 └──────────────┘
   ↑ 主: カード本体 click → うちの作品詳細（各話一覧・ⓘ出所・回遊）
```

- [↗]（副・外部）と ⓘ（出所）は**別位置**に置き、タップ領域が重ならないようにする（[↗]=右上隅、ⓘ=メタ行）。

### レスポンシブ注記

- **常設検索バー**は一覧上部に全幅で常時表示（ヘッダの縮小検索とは別）。**五十音ボタン**はその直下。
- **グリッド列数は画面幅で増える**: モバイル **2** ／ タブレット **3** ／ デスクトップ **4〜6** ／ **ウルトラワイド 7〜8+**。
- カードは**正方形(1:1)**。**card 最小幅を維持し引き伸ばさない**（max-width で中央に絞らず**列を足す**方向に拡張）。
- **モバイルは左サイドバーをドロワー/ボトムシート**に格納（[⚙ 絞り込み・並び]）。五十音はモバイルで横スクロール。

### ② 動線

- **常設検索バー → 右グリッドを同ページ更新**（`?q=…`）。
- **五十音ボタン（あ〜わ）→ その行（list.json `col_key`）の作品に絞る**（`?row=あ`）。**注記**: 完全な読みは無いため**行内の厳密な50音順ソートは不可**＝行で括るのは正確、行内は**タイトル文字列順フォールバック**。
- **左フィルタ（タグ・クール）・並び替え（勢い/累計再生数/新着）→ 同ページ更新**。件数・適用中チップを更新。クール併用で「今期の人気」等。
- **一覧カードの導線（主/副）**:
  - **主アクション**: カード本体 click → **うちの作品詳細**（各話一覧・ⓘ出所・回遊）。
  - **副アクション**: カード右上隅の **[↗] → ニコニコ公式シリーズページ `nicovideo.jp/series/<id>` に直行**（うちの詳細を経由しない外部遷移。↗で外部と明示）。
- すべて **URL クエリに反映**（例 `?q=…&row=は&tag=日常&cours=2026春&sort=hot&page=2`）。**無限スクロールにしない**。

## ?（出所）ツールチップ — フィールド→出所マップ

各ラベル横の `ⓘ` にマウスオーバーで以下を簡潔表示（監査 `validation/api-availability.md` のE列が元）。

| 表示項目 | ツールチップ文言（出所） |
|----------|------------------------|
| タイトル | カタログ(list.json)／シリーズAPI(nvapi)由来 |
| サムネ | シリーズAPI(nvapi)／各話は検索API(snapshot)由来 |
| タグ | 検索API(snapshot)の tags／categoryTags |
| 概要（第1話のあらすじ） | 検索API(snapshot)の第1話 description を流用・HTML除去（シリーズ要約ではない） |
| 各話リスト | シリーズAPI(nvapi v2 series)由来（話順・支店判定込み）＝主源 |
| 再生数 | 検索API(snapshot)の viewCounter（累計） |
| 勢いスコア | 計算（再生数÷投稿経過日数 等の近似・履歴蓄積なし） |
| 五十音 | カタログ(list.json)の col_key（読みの行・完全な読みは持たない） |
| クール | 投稿日(startTime)から判定（過去作は不確実→番組表/period補助 or 不明） |
| 支店判定 | チャンネル ID 2632720／シリーズの owner.channel.id=ch2632720 |

- ツールチップは**詳細ページ中心**に付与。一覧の「勢いスコア/再生数」など計算・注意のある項目にも付ける。

## ランキング方針（重要）

- **支店スコープの正確な週/月/急上昇デルタは API から直接取得不可**（`validation/api-availability.md`）。かつ**自前の `viewCounter` 時系列蓄積は採用しない**。
- 「人気・勢い」は**蓄積なしの近似スコア**（例 `viewCounter` ÷ 投稿経過日数。formula は L3）。**真の期間デルタではない＝「勢い（目安）」**として扱う。
- **新着話は comment/like/mylist が 4〜5 割しか値を持たない**（監査）→ **勢いは view 主体**、新着でエンゲージ指標は出さない/0 を隠す。
- **直接構築可**: 累計再生数順 ／ 新着順 ／ クール絞り（現行季）。正確な期間デルタ・個人のアーリー度系は別プロジェクト（視聴履歴拡張）＝スコープ外。

## 新着の2種（データ注記）

REQ-0010 は 2 系統。ビルド時に nvapi series（各話）／snapshot を束ねて両方を静的 JSON 化。

| 種別 | 定義 | カード単位 | 配置 |
|------|------|-----------|------|
| **新着シリーズ** | シリーズ初出日（各話 `startTime` の最小）が新しい | シリーズ | トップ／一覧(新着順) |
| **最新の動画** | 話の `startTime` 最大（既存作の新着話を含む） | 話 | トップ |

## REQ 対応（再掲）

- トップ: 検索/TOP10「すべて見る」=REQ-0001、Hot/人気TOP=REQ-0002、今期/クールから探す=REQ-0004、タグから探す=REQ-0008、新着/最近追加（シリーズ＋最新話）=REQ-0010、ダーク/ライト切替=REQ-0012。
- 一覧: ブラウズ/検索=REQ-0001、勢い/累計=REQ-0002、クール=REQ-0004、タグ=REQ-0008、新着=REQ-0010、五十音=REQ-0011。
- 作品詳細: 各話一覧/概要=REQ-0001、deep-link=REQ-0005、同じタグ回遊=REQ-0008。
