# 画面設計（情報設計レベル）

> L2（`foundation.md`）から分離した**画面（ページ）構成**。各ページに **①ワイヤー（配置＋大きさ・モバイル/デスクトップ/ウルトラワイド）**
> と **②動線** の 2 層を持たせる。細かい CSS・コンポーネント実装は L3 以降。
> **レスポンシブ必須**: モバイル / デスクトップ / ウルトラワイドの 3 段で挙動を注記する。

## 画面一覧（v1＝3画面）

| 画面 | 役割 | 対応 REQ |
|------|------|---------|
| **1. トップ** | 入口・発見。検索＋見せ場セクションから一覧へ送る | REQ-0001/0002/0004/0010 |
| **2. 一覧** | 探索の主戦場。左フィルタ＋右グリッドで絞り込み・並び替え | REQ-0001/0002/0003/0004/0008/0010 |
| **3. 作品(シリーズ)詳細** | 各話一覧＋公式 deep-link＋同タグ/ジャンルへ回遊 | REQ-0001/0005（＋回遊で 0003/0008） |

いずれも実行時は**事前生成した用途別 JSON を読むだけ**（API 直叩きなし）。**全状態は URL クエリで再現**（共有・ブックマーク・戻り位置が予測どおり）。

           │ カード click            └───────┬───────────────────────────┘
           │                                 │ カード click（シリーズ）
           ▼                                 ▼

### レスポンシブ注記

- **PC/ウルトラワイド: 横スクロール廃止**。各セクションは「その幅に収まる枚数を 1 行ベタ置き＋[すべて見る →]」。幅が広いほど 1 行枚数が増える。
- **モバイルのみ**見せ場を**横スワイプのカルーセル**にする（スナップ）。
- **ヒーロー検索**は PC では中央大 → スクロールで**ヘッダ縮小検索（sticky）**。モバイルでも上部に出す。
- **ウルトラワイド**は見せ場を**2 段組み（左右並び）**にするオプション可（上図）。

### ② 動線

- **入口 → トップ**。最初に目に入るのはヒーロー検索（検索ファースト）。
- **各見せ場の [すべて見る →] → 一覧**へ（その見せ場に対応する**フィルタ・並び替え状態を引き継ぐ**。例: 「今期2026春」→ 一覧をクール=2026春で開く／「人気・勢い順」→ 並び替え=勢い順）。
- **カード click → 作品詳細**（最新話カードは該当シリーズ詳細のその話に着地）。
- **検索入力 → 一覧**へ（`?q=…` 状態）。
- すべて **URL クエリに反映**。戻る/共有で同じ状態を再現。

### レスポンシブ注記

- **グリッド列数は画面幅で増える**: モバイル **2** ／ タブレット **3** ／ デスクトップ **4〜6** ／ **ウルトラワイド 7〜8+**。

### ② 動線

- **左フィルタ（ジャンル/タグ/クール）・並び替え（勢い/再生数/新着）→ 右グリッドを同ページ更新**（遷移しない）。件数・適用中チップを更新。
- **クール絞り＋並び替えの併用**で「今期の人気」等を表現（例 `?cours=2026春&sort=hot`）。
- **カード click → 作品詳細**。
- すべて **URL クエリに反映**（例 `?q=…&genre=…&tag=日常&cours=2026春&sort=hot&page=2`）。**無限スクロールにしない**＝戻り位置が予測どおり。

- **[← 戻る] → 直前のページ**（トップ or 一覧。URL で状態復元）。

## ランキング方針（重要）

検証（`validation/api-availability.md` 2回目）の結論を設計に反映する。

- **支店スコープの正確な週/月/急上昇デルタは、snapshot・公開ランキング API のどちらからも直接取得できない。**
- かつ **自前の `viewCounter` 時系列蓄積は採用しない**（保守・容量コストを避ける）。
- 代わりに **蓄積なしの近似スコアで「人気・勢い」を算出**する（＝トップ「人気・勢い順」／一覧「勢い順」）。
  - 例: **1日あたり再生数＝`viewCounter` ÷ 投稿からの経過日数**、または
    再生＋コメント＋いいね＋マイリストの累計に**経過時間の減衰**をかけた重み付き。
  - **具体 formula は L3 で確定。** これは**真の期間デルタではなく近似**である旨を UI/実装で正直に扱う（「勢い（目安）」等）。
- **直接構築できるもの**: 総合（累計再生数）順 ／ 新着（`startTime`）順 ／ クール絞り。
- **スコープ外**: 正確な期間デルタや「自分のアーリー度」系は**別プロジェクト（視聴履歴ブラウザ拡張）の領分**＝本ビューアの対象外。

## 新着の2種（データ注記）

REQ-0010 は 2 系統に割れる。ビルド時に **snapshot（話＝`contentId` 単位）を list.json の series id で束ねて**両方を静的 JSON 化する。

| 種別 | 定義 | カード単位 | 配置 |
|------|------|-----------|------|
| **新着シリーズ** | シリーズ初出日（束ねた各話 `startTime` の最小）が新しい | シリーズ | トップ／一覧(新着順) |
| **最新の動画** | 話の `startTime` 最大（既存作の新着話を含む） | 話 | トップ |

## REQ 対応（再掲）

- トップ: 検索/すべて見る=REQ-0001、人気・勢い=REQ-0002、クール=REQ-0004、新着（**新着シリーズ＋最新話の 2 系統**）=REQ-0010。
- 一覧: ブラウズ/検索=REQ-0001、勢い/累計=REQ-0002、ジャンル=REQ-0003、クール=REQ-0004、タグ=REQ-0008、新着=REQ-0010。
- 作品詳細: 各話一覧=REQ-0001、deep-link=REQ-0005、同タグ/ジャンル回遊=REQ-0003/0008。
