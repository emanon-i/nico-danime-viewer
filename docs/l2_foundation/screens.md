# 画面設計（情報設計レベル）

> L2（`foundation.md`）から分離した**画面（ページ）構成**。各ページに **①ワイヤー（配置＋大きさ・モバイル/デスクトップ/ウルトラワイド）**
> と **②動線** の 2 層を持たせる。細かい CSS・コンポーネント実装は L3 以降。
> **掲載項目はすべてデータ可用性監査（`validation/api-availability.md`）で裏取り済みのものだけ**。
> **`ⓘ`= 出所/意味ツールチップ**（平易な利用者向け文言）。**乱発しない＝各画面で最も目立つ 1 か所に集約**（カードの各フィールドや各話行ごとには付けない）。文言は本書末尾。
> **`⚙`= 設定/情報モーダル**（ヘッダ右上。設定＋情報。詳細は「ヘッダ（全ページ共通仕様）」「設定 / 情報モーダル」）。inline の `ⓘ` とは別物。
> **`🔍`= 検索ボタン**（ヘッダ常設・全画面のアイコン）。押すと検索バーが展開して自動フォーカス、Esc/× で閉じる。詳細は「検索（全ページ共通仕様）」。
> **パンくず**: トップ以外はヘッダ直下に表示。詳細は「パンくず（全ページ共通仕様）」。
>
> **実装状況**: 【設計・未実装】 — v1 全画面（トップ/一覧/詳細）は設計確定・実装フェーズ待ち。

## 1. トップ（入口・発見）

検索ファースト＋「考えられた順」のセクション積み（**カルーセル4段積みは廃止**）。上から: ①ヘッダ ②ヒーロー検索
③クイックアクセスのボタン列 ④人気シリーズ TOP10 ⑤最近追加・更新 ⑥クールから探す ⑦タグから探す。
各セクションの**順序と既定値の根拠**は本節末「設計根拠」に1行ずつ明記。カードは原則**シリーズ単位**。

- **ヘッダ**: ロゴ＋**🔍検索**＋**ダーク/ライト切替トグル（☀/🌙）**＋**⚙設定/情報**（右上は左→右で `🔍 → ☀/🌙 → ⚙`）。テーマ既定は OS の `prefers-color-scheme`、選択は
  `localStorage` に保存（**テーマ設定のみ＝個人化データではない**）。REQ-0012。
  **ヘッダの🔍はヒーローが見えている間は出さず、スクロールでヒーローが隠れてから出す**（同時に検索が 2 つ出ない）。
- **ヒーローに見出しテキストは置かない**（検索バーが主役）。**検索のコピー（「アニメ」でなく「作品/コンテンツ」寄り＝支店は舞台・ライブ・旧作も含む）は入力欄の placeholder「作品・タグで検索…」に集約**する。
- **検索バーとクイックアクセスの間に罫線（divider）は引かない**（検索→クイックアクセスを自然につなげる）。セクション区切り線は④以降の本文ショーケースから。

### クイックアクセスのプリセット・デフォルト

各ボタンは**一覧の特定プリセット状態（URL クエリ）**を開く。

| 入口                     | フィルタ                                          | 並び替え(デフォルト)             | 備考                                                                                           |
| ------------------------ | ------------------------------------------------- | -------------------------------- | ---------------------------------------------------------------------------------------------- |
| **今期**                 | クール=現行季（period配線・`foundation.md` §1.2） | 勢い順                           | 今季の話題順                                                                                   |
| **人気**                 | なし                                              | 累計再生数順                     | **全期間のみ＝期間セレクタ無し**（履歴を貯めないので月間/週間は作れない。「今」は Hot が担当） |
| **Hot**                  | なし                                              | 勢い順（前日比 delta＋velocity） | 「今の伸び」目安                                                                               |
| **ランダムタグA**        | タグ=タグ辞書からサンプル（ページロード時決定）   | 勢い順                           | `?tag=<sampled>`。毎ロード異なるタグ → 一覧タグ絞り（発見性）                                  |
| **ランダムタグB**        | タグ=タグ辞書からサンプル（A とは重複しないもの） | 勢い順                           | `?tag=<sampled>`。A と異なるタグを選ぶ                                                         |
| 一覧で探す（独立ボタン） | なし                                              | 人気(累計)順                     | `?sort=views`。ヒーロー検索バー直下の独立ボタン（クイックアクセスとは**別物**）                |

> **クイックアクセスは機能的に重複しない入口（厳選5）のみ**。新着は⑤「最近追加・更新」セクション＋一覧の新着ソートで担保。五十音は一覧ページのナビゲーションで担保（クイックアクセスには置かない）。ランダムタグ×2は発見性のためにページロード時にタグ辞書からサンプリングする（引き直しは⑦タグから探すの🔀ボタン）。

### レスポンシブ注記

- **ヘッダのテーマ切替（☀/🌙）は全幅で表示**。テーマ既定は `prefers-color-scheme`、選択は `localStorage` 保存（即時反映・遷移なし）。
  **ヘッダ🔍はヒーローがスクロールで隠れてから出す**（トップは検索が同時に 2 本にならない）。
- **横スクロールの多用を避ける**（発見系の全部を横カルーセルにしない）。ただし**人気シリーズ TOP10 のような有限のランキング帯は PC/ウルトラワイドでも横スクロール可**（横に伸びるのが自然・続きは [すべて見る →] で一覧へ）。**スクロールバーは非表示・スワイプ移動**。ウルトラワイドは⑤⑥⑦を 2 段組みに。
- **④TOP10 カードは看板として他セクションより大きく**（`.card-rail .series-card` 幅 `clamp(264px,32vw,360px)`／モバイルは約 72vw で「次がチラ見え」）。⑤の list-row（小）との**大小差でメリハリ**を出す。
- **一覧の作品グリッドは縦グリッド**（ここは横スクロールにしない）。TOP10 はモバイルでも横スワイプ、クール/五十音はモバイルで横スクロール。
- **③クイックアクセスはヒーロー検索の直下**に置く。**長い場合は横スワイプ（可視スクロールバーを出さない・はみ出しは `→` で示唆）**＝モバイル/デスクトップ/ウルトラワイドとも。**「一覧で探す」ボタンは検索バー直下・クイックアクセスの上に独立配置**（`?sort=views`）。
- **⑦タグから探す**は **Hotのタグ／人気のタグ／ランダムタグ＋定番タグ**のチップ群。各 click → 一覧のタグ絞り（`?tag=…`）。いずれも**正規化済みタグ**（dアニメキュレーション正規化含む）から生成。長い行は横スワイプ（バー非表示）、ランダムは [🔀] で引き直し。
- カードは**16:9 で統一**（`--card-aspect`）。**素サムネは 4:3（360×270・`.L`）だが、中身は 16:9 映像を上下黒帯でレターボックスした焼き込み**。**16:9＋`object-fit:cover` で上下の黒帯（各 1/8）を正確にトリミング**して実コンテンツだけを全面表示する（黒帯はソース由来＝こちらの比率設定で除去）。**カードに ⓘ は付けない**（アイコンは ♥/✓/[↗] のみ）。詳細は `design-system.md` §6.4。

### 設計根拠（なぜこの順・この既定か）

- **②ヒーロー検索を最上部**: カタログが広く目的来訪が多い → 検索ファースト。コピーを「作品/コンテンツ」にしたのは支店が舞台/ライブ/旧作も含むため。
- **③クイックアクセス厳選5**: 「今期/人気/Hot＋ランダムタグ×2」を対等な入口として並置。新着は⑤最近追加・更新セクションで担保、五十音は一覧ページのナビゲーションで担保（クイックアクセスに入れると役割が曖昧になるため除外）。ランダムタグ×2は発見性のためにページロード時にタグ辞書からサンプリングする。
- **④人気TOP10ショーケース**: まず看板（定番の安心）を見せて回遊を喚起。**累計再生数順**＝予測どおりで安定。**カードは他セクション（list-row 等）より明確に大きくして主役感・メリハリを出す**（看板帯。`design-system.md` §6.1）。
- **⑤最近追加・更新**: 「変化」を提示して再訪動機を作る。**2 種を別の見た目の list-row で出す（紛らわしさ防止＝異なるものは異なる見た目に）**:
  - **新着シリーズ型**: サムネ＋**「シリーズ」バッジ（layers アイコン）**＋作品名＋**「[film]N話」**（各話数＝`episodes` 件数・「全」を省き単位語「話」は残す）。主＝うちの詳細（`?series=`）／副＝**↗ 公式シリーズ `/series/<id>`**。
  - **最新の動画（各話）型**: サムネ（**▶ オーバーレイ**）＋**「第N話」バッジ**（**タイトル表記の第N話を優先**・無ければ nvapi `episode_no`）＋話タイトル（バッジと重複する第N話は本文から除去）＋**再生数・投稿日(M/D)**。主＝**公式 watch（外部）**／副＝**↗ 公式 watch `/watch/<contentId>`**。
  - **2 列（左=シリーズ型／右=各話型）は横並びのとき整列**: 件数を揃え（最大 5）、見出し同高・各行同高で**ベースラインを揃える**。モバイルは縦積み。
- **⑥クール→⑦タグの順**: 季節は粒度が粗く入りやすい上位、タグはより能動的・細粒度なので下。**タグに舞台/ライブ等の非アニメ**を含め、アニメ以外の発見を掬う。
- **Hot と 人気TOP の役割分離**: Hot=「**今の勢い（前日比 delta＋velocity）**」、人気TOP=「**全期間の定番（累計）**」。正確な週/月デルタを持てない制約を、2 ボタンの役割分担で正直に表現（この違いは ④の ⓘ 1 か所で説明）。
- **人気TOP に期間セレクタを置かない理由**: 無制限履歴は持たない（prev 1 スロットのみ）→ 月間/週間の正確ランキングは作れない。「今」の軸は Hot が担うので、TOP は全期間累計に限定（できない期間を UI に出さない＝予測どおり）。
- **ダーク/ライト**: 既定は OS 追従（`prefers-color-scheme`）、選択は `localStorage`。**テーマ設定のみで個人化データではない**（視聴履歴等は扱わない方針と矛盾しない）。
- **検索を 1 本に**: トップはヒーロー検索が主役、ヘッダ🔍はヒーローが隠れてから出す。**画面に検索入力が同時に 2 つ出ない**（迷わせない）。

### ② 動線

- **入口 → トップ**。最初に目に入るのはヒーロー検索（検索ファースト）。
- **ヒーロー検索入力 → 一覧**へ（`?q=…`）。スクロール後は**ヘッダ🔍 → 展開フォーカス → 同じく一覧へ**。
- **「一覧で探す」**（独立ボタン）: **ヒーロー検索バーの直下**に置く（`?sort=views`）。クイックアクセスとは別物。キーワードが思いつかない・とにかく全作品を見たい利用者向けの明示入口。
- **クイックアクセス [今期][人気][Hot][タグA][タグB] → 一覧の各プリセット状態／タグ絞り**（上表の通り。URL クエリ。例 今期=`?cours=current&sort=hot`、人気=`?sort=views`、ランダムタグ=`?tag=<sampled>`）。Hot/人気の意味の違いは**④人気TOP10 の ⓘ 1 か所**で説明（各ボタン横には ⓘ を並べない）。**新着は⑤セクション・五十音は一覧ナビ**で担保。
- **TOP10 カード → 作品詳細**（主）／カード隅 **[↗] → 公式シリーズ**（副・外部）。
- **最近追加の各行 → シリーズ型は詳細（主）＋ ↗ 公式シリーズ（副）／各話型は公式 watch（主）＋ ↗ 公式 watch（副）**。**各行に ↗（外部明示）を常時表示**（`rel=noopener noreferrer`・別タブ）。
- **クールボタン → 一覧のクール絞り**（`?cours=2026春`）。**タグから探す（Hot/人気/ランダム/定番のタグチップ）→ 一覧のタグ絞り**（`?tag=…`）。Hot/人気は上位作品の頻出タグ集計、ランダムはタグ辞書からサンプル（[🔀] で引き直し）。
- **テーマトグル（☀/🌙）→ 即時反映＋`localStorage` 保存**（遷移なし）。**⚙ → 設定/情報モーダル**。
- 状態は **URL クエリで再現**（テーマのみ `localStorage`）。

### カード内訳（主/副アクション・全幅共通）

16:9 カードは**主アクション＝カード本体、副アクション＝右上隅の小さな [↗] オーバーレイ**を併存させる。**カードに ⓘ は付けない**。

```
 ┌──────────────┐
 │♥✓ thumb   [↗]│ ← 左上: ♥お気に入り / ✓見た（トグル）  右上: 副[↗]

   ↑ 主: カード本体 click → うちの作品詳細（各話一覧・回遊）
```

- **♥/✓（左上）・[↗]（右上）は別位置**に置き、タップ領域が重ならないようにする。
- **♥ お気に入り／✓ 見た**＝トグル。状態は **localStorage**（「id＋フラグ」程度の小さな JSON・クライアント側・サーバに出さない）。視聴履歴拡張（別プロジェクト）が入っていれば「✓ 見た」は自動反映できる。大規模化（拡張から視聴履歴を大量取り込み等）する場合は IndexedDB を再検討。

### レスポンシブ注記

- **一覧は最上部に検索バーを置く（維持）**。ヘッダ🔍ボタンからも検索でき、**入力バーは一覧の 1 本だけなので重複しない**。**五十音ボタン**は検索バーの下。
- **グリッド列数は画面幅で増える**: モバイル **2** ／ タブレット **3** ／ デスクトップ **4〜6** ／ **ウルトラワイド 7〜8+**。
- カードは**16:9**（`--card-aspect`・ソースの焼き込み黒帯はトリミング済み）。**card 最小幅を維持し引き伸ばさない**（max-width で中央に絞らず**列を足す**方向に拡張）。
- **モバイルは左サイドバー（タグ/クール/並び替え）を「絞り込み・並び」ボタン → ドロワー（下/横）に格納**し、選択後に閉じる（常時表示しない）。**デスクトップ/ウルトラワイドは左サイドバー常駐**。五十音はモバイルで横スクロール。パンくずは先頭省略「…」。

### ② 動線

- **最上部検索バー（またはヘッダ🔍）→ 右グリッドを同ページ更新**（`?q=…`）。
- **五十音ボタン（あ〜わ）→ その行（読みの行）の作品に絞る**（`?row=あ`）。**注記**: 完全な読みは無いため**行内の厳密な50音順ソートは不可**＝行で括るのは正確、行内は**タイトル文字列順フォールバック**。
- **並び替え「五十音順」**（`?sort=kana`）: 読みの行（あ→わ）順に並べる。行単位は正確だが、**完全な読み(yomi)が無いため行内は厳密な50音順にならずタイトル文字列順フォールバック**（＝「だいたい五十音」）。
- **左フィルタ（タグ・クール）・並び替え（勢い/累計再生数/新着/五十音順）→ 同ページ更新**。件数・適用中チップ・**パンくずの状態表記**を更新。クール併用で「今期の人気」等。勢い順/累計の意味は**「並び替え」見出しの ⓘ 1 か所**で説明。
  - **タグ**（`?tag=…`）は**フラット 1 系統**。dアニメのキュレーションタグ（接頭/接尾両型）を正規化して通常タグと同列に扱う（品質が高いので候補の優先表示はしてよいが、別 facet にはしない）。
- **一覧カードの導線（主/副）**:
  - **主アクション**: カード本体 click → **うちの作品詳細**。
  - **副アクション**: カード右上隅の **[↗] → ニコニコ公式シリーズページ `nicovideo.jp/series/<id>` に直行**（うちの詳細を経由しない外部遷移。↗で外部と明示）。
- **お気に入り/見た**: カード左上の **♥（お気に入り）/✓（見た）トグル → localStorage に保存**（クライアント側・サーバ無し・遷移なし）。サイドバー/ドロワーの **「♥お気に入りだけ／✓未視聴だけ」**で絞り込み可（任意・軽量）。
- **ロゴ click → トップ**（全ページ共通）。**パンくず**: `ホーム` → トップ、`一覧：<状態>` → その状態の一覧。すべて **URL クエリに反映**（例 `?q=…&row=は&tag=日常&cours=2026春&sort=kana&page=2`）。**無限スクロールにしない**。

支店ラインナップには**配信停止っぽい作品も混在し、そのまま扱う**。各話/メタが取れない・公式ページが無い場合は、欠損を**正直に出す**（壊さない・空白で誤魔化さない）。

│ [ロゴ]                                      [🔍] [☀/🌙] [⚙]  │ ヘッダ(🔍/⚙設定/情報/テーマ)
├──────────────────────────────────────────────────────────────┤
│ ホーム / 一覧 / 《作品名》                                       │ パンくず
├──────────────────────────────────────────────────────────────┤
│ ┌──────────┐   作品タイトル（取れたメタだけ表示）   [♥][✓]     │ 取れるメタのみ
│ │ thumb無し │   タグ: （取得できた分のみ／無ければ非表示）      │
│ │  or 代替  │                                                  │
│ └──────────┘                                                  │
├──────────────────────────────────────────────────────────────┤
│  ⚠ この作品の配信情報は取得できませんでした（配信終了の可能性）   │ ← empty メッセージ
│    ・各話一覧: 取得できませんでした                              │ 各話が空ならその旨
│    ・公式ページ: [▶ 公式シリーズページ →]（生存時のみ表示）       │ リンクは生存時だけ
├──────────────────────────────────────────────────────────────┤
│ ▸ 同じタグへ [日常 →] …（タグが取れていれば回遊は生かす）        │ 回遊は可能なら維持
└──────────────────────────────────────────────────────────────┘

- **モバイル/ウルトラワイド**も同じ方針（縦積み／中央 max-width）。**取れたメタだけ表示・取れない部分は「取得できませんでした」と明示・公式リンクは生存時のみ**。
- 判定: nvapi series が空/失敗、または公式ページ不在。**変更検知（`foundation.md` §5.4）とは別**（個別作品の欠損は fail させず、その作品だけ empty 表示）。

- **トップ/一覧のカード click → ここに着地**。

## 検索（全ページ共通仕様）

- **ヘッダの🔍は全画面共通のボタン**。押すと**検索バーが展開して自動フォーカス**、`Esc` か `×` で閉じる。**ヘッダに入力バーを常時は置かない**（アイコンのみ）。
  ヘッダがボタンなので、ページ内の検索バーと**同時に出ても入力欄は 1 本**（重複しない）。
- **各ページの検索入力**:
  - **トップ**: ヒーロー検索バー（主役）。ヘッダ🔍はヒーローが**スクロールで隠れてから**出す。

## パンくず（全ページ共通仕様）

- **トップ（ルート）には出さない。** トップ以外はヘッダ直下（コンテンツ最上部）に表示。
- **一覧**: `ホーム / 一覧` ＋**適用中の状態を反映**。
  例: `ホーム / 一覧：今期 2026春` ／ `ホーム / 一覧：タグ「日常」` ／ `ホーム / 一覧：検索「ぼっち」`。
  プリセット（今期/人気/Hot）経由ならその名前を出す（例 `ホーム / 一覧：Hot`）。ランダムタグ経由は `ホーム / 一覧：タグ「<タグ名>」`。
- **詳細**: `ホーム / 一覧 / 《作品名》`。特定フィルタ経由なら経路を反映（例 `ホーム / 一覧：タグ「日常」/ 《作品名》`）。
- **各セグメントはクリックで該当階層へ**（`ホーム`→トップ、`一覧：<状態>`→その状態の一覧）。**URL クエリ状態と整合**＝戻り先が予測どおり。
- **役割の整理**: 階層移動はパンくずに一本化（ヘッダに別途の「← 戻る」ボタンは置かない＝重複させない）。一覧の `[← 前][次 →]` はページング。
- **モバイル**: 長いパンくずは**先頭を省略**（`… / 一覧：日常 / 作品名`）。末尾（現在地）と直近の親を優先表示。

## ランキング方針（重要）

- **支店スコープの正確な週/月の急上昇（期間デルタ）は API から直接取得不可**（`validation/api-availability.md`）。**無制限の時系列蓄積はしない**が、`prev_view_counter`（1 スロット bounded）で前日比 delta は出す。
- 「人気・勢い」は**前日比 delta＋velocity＋recency のブレンド**（**定義は `foundation.md` §1.2 / [`db-design.md`](db-design.md)**。delta は翌日から有効・日次解像度。係数は L3）。**正確な週/月デルタではない＝「勢い（目安）」**として扱い、意味は**各画面 1 か所の ⓘ**で説明。
- **新着話は comment/like/mylist が 4〜5 割しか値を持たない**（監査）→ **勢いは view 主体**、新着でエンゲージ指標は出さない/0 を隠す。
- **直接構築可**: 累計再生数順 ／ 新着順 ／ クール絞り（現行季）。正確な期間デルタ・個人のアーリー度系は別プロジェクト（視聴履歴拡張）＝スコープ外。

## 新着の2種（データ注記）

REQ-0010 は 2 系統。ビルド時に nvapi series（各話）／snapshot を束ねて両方を静的 JSON 化。

| 種別             | 定義                                              | カード単位 | 行の見た目（list-row 型）                                                                    | 配置                 |
| ---------------- | ------------------------------------------------- | ---------- | -------------------------------------------------------------------------------------------- | -------------------- |
| **新着シリーズ** | シリーズ初出日（各話 `startTime` の最小）が新しい | シリーズ   | **シリーズ型**: layers バッジ＋作品名＋**[film]N話**＋ ↗ `/series/<id>`                      | トップ／一覧(新着順) |
| **最新の動画**   | 話の `startTime` 最大（既存作の新着話を含む）     | 話         | **各話型**: ▶＋**第N話**バッジ＋話タイトル＋**[play]再生数 ・ [clock]M/D**＋ ↗ `/watch/<id>` | トップ               |

- **[film]N話**＝シリーズの各話数（`episodes` テーブル件数。export 時に `works.json` の `episodeCount` へ）。「全」を省き単位語「話」は残す（`design-system.md` §8.2）。
- **第N話**＝**話タイトル中の「第N話」表記を優先**（公式表記と一致）、無ければ nvapi `episode_no`（番号がずれることがあるため後者は後順）。
- **最新の動画は resolved（`so…` 解決済み）のみ表示**（watch リンクが張れない rss_only は出さない）。サムネは `episodes.thumbnail_url` を `resolved_content_id` で join（`.L` 昇格・16:9 cover）。取得不能時は **▶ プレースホルダ**。
