# 画面設計（情報設計レベル）

> L2（`foundation.md`）から分離した**画面（ページ）構成**。各ページに **①ワイヤー（配置＋大きさ・モバイル/デスクトップ/ウルトラワイド）**
> と **②動線** の 2 層を持たせる。細かい CSS・コンポーネント実装は L3 以降。
> **掲載項目はすべてデータ可用性監査（`validation/api-availability.md`）で裏取り済みのものだけ**。
> **`ⓘ`= 出所ツールチップ**（各ラベル横の「?」。マウスオーバーで**利用者向けの平易な言葉**で取得元・近似可否を表示。文言は本書末尾）。
> **`🔍`= 検索ボタン**（ヘッダ常設・全画面のアイコン）。押すと検索バーが展開して自動フォーカス、Esc/× で閉じる。詳細は「検索（全ページ共通仕様）」。
> **パンくず**: トップ以外はヘッダ直下に表示。詳細は「パンくず（全ページ共通仕様）」。

## 画面一覧（v1＝3画面）

| 画面 | 役割 | 対応 REQ |
|------|------|---------|
| **1. トップ** | 入口・発見。ヒーロー検索＋クイックアクセス＋TOP10＋最近追加＋クール/タグ入口から一覧へ送る。テーマ切替 | REQ-0001/0002/0004/0008/0010/0012 |
| **2. 一覧** | 探索の主戦場。最上部検索バー＋五十音＋左フィルタ（タグ/クール）＋右グリッド＋♥/✓ | REQ-0001/0002/0003/0004/0008/0010/0011/0015 |
| **3. 作品(シリーズ)詳細** | 各話一覧＋公式 deep-link＋関連シリーズ/続編＋♥/✓＋タグチップ回遊（欠損は empty 表示） | REQ-0001/0005/0014/0015（＋回遊で 0003/0008） |

いずれも実行時は**事前生成した用途別 JSON を読むだけ**（API 直叩きなし）。**全状態は URL クエリで再現**（共有・ブックマーク・戻り位置が予測どおり）。
**一覧の「タグ」フィルタはフラット 1 系統**。dアニメのキュレーションタグ（`dアニメ` マーカー付き・接頭/接尾両型）は**正規化して通常タグと同じ平面で扱う**（監査: `genre` 欄は 99.95%「アニメ」一色で使えないため使わない）。

## 1. トップ（入口・発見）

検索ファースト＋「考えられた順」のセクション積み（**カルーセル4段積みは廃止**）。上から: ①ヘッダ ②ヒーロー検索
③クイックアクセスのボタン列 ④人気シリーズ TOP10 ⑤最近追加・更新 ⑥クールから探す ⑦タグから探す。
各セクションの**順序と既定値の根拠**は本節末「設計根拠」に1行ずつ明記。カードは原則**シリーズ単位**。

- **ヘッダ**: ロゴ＋ナビ＋**ダーク/ライト切替トグル（☀/🌙）**。既定は OS の `prefers-color-scheme`、ユーザー選択は
  `localStorage` に保存（**テーマ設定のみ＝個人化データではない**）。REQ-0012。
  **ヘッダの🔍はヒーローが見えている間は出さず、スクロールでヒーローが隠れてから出す**（同時に検索が 2 つ出ない）。
- **ヒーロー検索のコピー**は「アニメ」でなく**「作品/コンテンツ」**寄り（支店は舞台・ライブ・旧作も含むため）。

### クイックアクセスのプリセット・デフォルト

各ボタンは**一覧の特定プリセット状態（URL クエリ）**を開く。

| 入口 | フィルタ | 並び替え(デフォルト) | 備考 |
|------|---------|---------------------|------|
| **今期** | クール=現行季（`startTime` から自動判定） | 勢い順 | 今季の話題順 |
| **新着** | なし | 新着シリーズ順（シリーズ初出降順） | タイトル単位の新規 |
| **Hot** | なし | 勢い順（蓄積なし近似） | 「今の伸び」目安 |
| **人気TOP** | なし | 累計再生数順 | **全期間のみ＝期間セレクタ無し**（履歴を貯めないので月間/週間は作れない。「今」は Hot が担当） |
| 一覧 直アクセス | なし | 人気(累計)順 | 予測どおり重視（安定） |

### レスポンシブ注記

- **ヘッダのテーマ切替（☀/🌙）は全幅で表示**。テーマ既定は `prefers-color-scheme`、選択は `localStorage` 保存（即時反映・遷移なし）。
  **ヘッダ🔍はヒーローがスクロールで隠れてから出す**（トップは検索が同時に 2 本にならない）。
- **PC/ウルトラワイド: 横スクロール廃止**。TOP10 ショーケースは「幅に収まる枚数を 1 行ベタ置き＋[すべて見る →]」。ウルトラワイドは⑤⑥⑦を 2 段組みに。
- **モバイルのみ** TOP10 を**横スワイプ**。クールは横スクロール。
- **③クイックアクセスはヒーロー検索の直下**に置く。**長い場合は横スワイプ（可視スクロールバーを出さない・はみ出しは `→` で示唆）**＝モバイル/デスクトップ/ウルトラワイドとも。
- **⑦タグから探す**は **Hotのタグ／人気のタグ／ランダムタグ＋定番タグ**のチップ群。各 click → 一覧のタグ絞り（`?tag=…`）。いずれも**正規化済みタグ**（dアニメキュレーション正規化含む）から生成。長い行は横スワイプ（バー非表示）、ランダムは [🔀] で引き直し。
- カードは**正方形(1:1)**で統一。

### 設計根拠（なぜこの順・この既定か）

- **②ヒーロー検索を最上部**: カタログが広く目的来訪が多い → 検索ファースト。コピーを「作品/コンテンツ」にしたのは支店が舞台/ライブ/旧作も含むため。
- **③今期を単独先頭にせず横並びボタンで両立**: 「今期/新着/Hot/人気TOP」は**対等な入口**。一つを固定先頭にすると他の動機が埋もれる → ボタン列で並置。
- **④人気TOP10ショーケース**: まず看板（定番の安心）を見せて回遊を喚起。**累計再生数順**＝予測どおりで安定。
- **⑤最近追加・更新**: 「変化」を提示して再訪動機を作る（新着シリーズ＝新規タイトル、最新話＝既存作の更新）。
- **⑥クール→⑦タグの順**: 季節は粒度が粗く入りやすい上位、タグはより能動的・細粒度なので下。**タグに舞台/ライブ等の非アニメ**を含め、アニメ以外の発見を掬う。
- **Hot と 人気TOP の役割分離**: Hot=「**今の勢い（蓄積なし近似）**」、人気TOP=「**全期間の定番（累計）**」。期間デルタを持てない制約を、2 ボタンの役割分担で正直に表現。
- **人気TOP に期間セレクタを置かない理由**: 履歴を蓄積しない方針 → 月間/週間ランキングは作れない。「今」の軸は Hot が担うので、TOP は全期間累計に限定（できない期間を UI に出さない＝予測どおり）。
- **ダーク/ライト**: 既定は OS 追従（`prefers-color-scheme`）、選択は `localStorage`。**テーマ設定のみで個人化データではない**（視聴履歴等は扱わない方針と矛盾しない）。
- **検索を 1 本に**: トップはヒーロー検索が主役、ヘッダ🔍はヒーローが隠れてから出す。**画面に検索入力が同時に 2 つ出ない**（迷わせない）。

### ② 動線

- **入口 → トップ**。最初に目に入るのはヒーロー検索（検索ファースト）。
- **ヒーロー検索入力 → 一覧**へ（`?q=…`）。スクロール後は**ヘッダ🔍 → 展開フォーカス → 同じく一覧へ**。
- **クイックアクセス [今期][新着][Hot][人気TOP] → 一覧の各プリセット状態**（上表の通り。URL クエリ。例 今期=`?cours=現行季&sort=hot`、人気TOP=`?sort=views`）。
- **TOP10／最近追加のカード → 作品詳細**（主）／カード隅 **[↗] → 公式シリーズ**（副・外部）。
- **クールボタン → 一覧のクール絞り**（`?cours=2026春`）。**タグから探す（Hot/人気/ランダム/定番のタグチップ）→ 一覧のタグ絞り**（`?tag=…`）。Hot/人気は上位作品の頻出タグ集計、ランダムはタグ辞書からサンプル（[🔀] で引き直し）。
- **テーマトグル（☀/🌙）→ 即時反映＋`localStorage` 保存**（遷移なし）。
- 状態は **URL クエリで再現**（テーマのみ `localStorage`）。

### カード内訳（主/副アクション・全幅共通）

正方形カードは**主アクション＝カード本体、副アクション＝右上隅の小さな [↗] オーバーレイ**を併存させる。

```
 ┌──────────────┐
 │♥✓ thumb   [↗]│ ← 左上: ♥お気に入り / ✓見た（トグル）  右上: 副[↗]
 │ (1:1)        │      ♥/✓ は IndexedDB（クライアント側・サーバ無し）
 │              │      [↗] → ニコニコ公式シリーズ /series/<id>（外部・詳細を経由しない）
 │ 作品タイトル   │
 │ 勢い/再生 ⓘ   │ ← ⓘ 出所ツールチップ（メタ行）
 └──────────────┘
   ↑ 主: カード本体 click → うちの作品詳細（各話一覧・ⓘ出所・回遊）
```

- **♥/✓（左上）・[↗]（右上）・ⓘ（メタ行）は別位置**に置き、タップ領域が重ならないようにする。
- **♥ お気に入り／✓ 見た**＝トグル。状態は **IndexedDB**（クライアント側・サーバに出さない）。視聴履歴拡張（別プロジェクト）が入っていれば「✓ 見た」は自動反映できる。

### レスポンシブ注記

- **一覧は最上部に検索バーを置く（維持）**。ヘッダ🔍ボタンからも検索でき、**入力バーは一覧の 1 本だけなので重複しない**。**五十音ボタン**は検索バーの下。
- **グリッド列数は画面幅で増える**: モバイル **2** ／ タブレット **3** ／ デスクトップ **4〜6** ／ **ウルトラワイド 7〜8+**。
- カードは**正方形(1:1)**。**card 最小幅を維持し引き伸ばさない**（max-width で中央に絞らず**列を足す**方向に拡張）。
- **モバイルは左サイドバー（タグ/クール/並び替え）を「絞り込み・並び」ボタン → ドロワー（下/横）に格納**し、選択後に閉じる（常時表示しない）。**デスクトップ/ウルトラワイドは左サイドバー常駐**。五十音はモバイルで横スクロール。パンくずは先頭省略「…」。

### ② 動線

- **最上部検索バー（またはヘッダ🔍）→ 右グリッドを同ページ更新**（`?q=…`）。
- **五十音ボタン（あ〜わ）→ その行（読みの行）の作品に絞る**（`?row=あ`）。**注記**: 完全な読みは無いため**行内の厳密な50音順ソートは不可**＝行で括るのは正確、行内は**タイトル文字列順フォールバック**。
- **並び替え「五十音順」**（`?sort=kana`）: 読みの行（あ→わ）順に並べる。行単位は正確だが、**完全な読み(yomi)が無いため行内は厳密な50音順にならずタイトル文字列順フォールバック**（＝「だいたい五十音」）。
- **左フィルタ（タグ・クール）・並び替え（勢い/累計再生数/新着/五十音順）→ 同ページ更新**。件数・適用中チップ・**パンくずの状態表記**を更新。クール併用で「今期の人気」等。
  - **タグ**（`?tag=…`）は**フラット 1 系統**。dアニメのキュレーションタグ（接頭/接尾両型）を正規化して通常タグと同列に扱う（品質が高いので候補の優先表示はしてよいが、別 facet にはしない）。
- **一覧カードの導線（主/副）**:
  - **主アクション**: カード本体 click → **うちの作品詳細**（各話一覧・ⓘ出所・回遊）。
  - **副アクション**: カード右上隅の **[↗] → ニコニコ公式シリーズページ `nicovideo.jp/series/<id>` に直行**（うちの詳細を経由しない外部遷移。↗で外部と明示）。
- **お気に入り/見た**: カード左上の **♥（お気に入り）/✓（見た）トグル → IndexedDB に保存**（クライアント側・サーバ無し・遷移なし）。サイドバー/ドロワーの **「♥お気に入りだけ／✓未視聴だけ」**で絞り込み可（任意・軽量）。
- **ロゴ click → トップ**（全ページ共通）。**パンくず**: `ホーム` → トップ、`一覧：<状態>` → その状態の一覧。すべて **URL クエリに反映**（例 `?q=…&row=は&tag=日常&cours=2026春&sort=kana&page=2`）。**無限スクロールにしない**。

## ヘッダ（全ページ共通仕様）

- **ロゴ**: クリックで**トップへ戻る**（全ページ共通）。
- **🔍 検索ボタン**: 「検索（全ページ共通仕様）」を参照。
- **☀/🌙 テーマ切替**: 既定 `prefers-color-scheme` ＋ 選択を `localStorage`（テーマ設定のみ）。
- 階層移動は**パンくず**（「パンくず（全ページ共通仕様）」）。ヘッダに別途の「← 戻る」は置かない。

## 検索（全ページ共通仕様）

- **ヘッダの🔍は全画面共通のボタン**。押すと**検索バーが展開して自動フォーカス**、`Esc` か `×` で閉じる。**ヘッダに入力バーを常時は置かない**（アイコンのみ）。
  ヘッダがボタンなので、ページ内の検索バーと**同時に出ても入力欄は 1 本**（重複しない）。
- **各ページの検索入力**:
  - **トップ**: ヒーロー検索バー（主役）。ヘッダ🔍はヒーローが**スクロールで隠れてから**出す。
  - **一覧**: **最上部の検索バー（維持）＋ヘッダ🔍**。バーは 1 本なので重複しない。
  - **詳細**: ヘッダ🔍のみ（コンテンツ側のバーは置かない）。
- 入力確定 → **一覧の検索結果**（`?q=…`）。トップ・詳細で打った場合も一覧へ遷移。状態は URL クエリで共有・再現可能。

## パンくず（全ページ共通仕様）

- **トップ（ルート）には出さない。** トップ以外はヘッダ直下（コンテンツ最上部）に表示。
- **一覧**: `ホーム / 一覧` ＋**適用中の状態を反映**。
  例: `ホーム / 一覧：今期 2026春` ／ `ホーム / 一覧：タグ「日常」` ／ `ホーム / 一覧：検索「ぼっち」`。
  プリセット（今期/新着/Hot/人気TOP）経由ならその名前を出す（例 `ホーム / 一覧：Hot`）。
- **詳細**: `ホーム / 一覧 / 《作品名》`。特定フィルタ経由なら経路を反映（例 `ホーム / 一覧：タグ「日常」/ 《作品名》`）。
- **各セグメントはクリックで該当階層へ**（`ホーム`→トップ、`一覧：<状態>`→その状態の一覧）。**URL クエリ状態と整合**＝戻り先が予測どおり。
- **役割の整理**: 階層移動はパンくずに一本化（ヘッダに別途の「← 戻る」ボタンは置かない＝重複させない）。一覧の `[← 前][次 →]` はページング。
- **モバイル**: 長いパンくずは**先頭を省略**（`… / 一覧：日常 / 作品名`）。末尾（現在地）と直近の親を優先表示。

## ?（出所）ツールチップ — 利用者向け文言

各ラベル横の `ⓘ` にマウスオーバーで、**平易な言葉で「どこからの情報か・近似かどうか」だけ**を表示する。
**API 内部名・機構用語（フィールド名やチャンネル ID 等）は出さない。** 開発者向けの「フィールド→取得源」対応表は
`foundation.md`（§1.2 の役割分担）／`validation/api-availability.md`（出所マップ）にある。

| 表示項目 | ツールチップ文言（利用者向け） |
|----------|------------------------------|
| タイトル | ニコニコの作品情報より |
| サムネ | ニコニコの作品情報より |
| タグ | ニコニコの作品タグより（ジャンル等のタグを含む） |
| 概要 | 第1話のあらすじを表示しています |
| 各話リスト | ニコニコのシリーズ情報より |

| 五十音 | 作品の読みの行（あ〜わ）で振り分けています |
| クール | ニコニコの放送時期情報より |
| 関連シリーズ/続編 | 作品のタグから推定（取れた範囲で表示しています） |

- ツールチップは**詳細ページ中心**に付与。一覧の「勢い／再生数」など注意のある項目にも付ける。

## ランキング方針（重要）

- **支店スコープの正確な週/月/急上昇デルタは API から直接取得不可**（`validation/api-availability.md`）。かつ**自前の `viewCounter` 時系列蓄積は採用しない**。
- 「人気・勢い」は**蓄積なしの近似スコア**（**定義は `foundation.md` §1.2**＝シリーズ合算再生数 ÷ 公開からの経過日数＝1日あたり平均ペース。係数は L3）。**真の期間デルタではない＝「勢い（目安）」**として扱う。
- **新着話は comment/like/mylist が 4〜5 割しか値を持たない**（監査）→ **勢いは view 主体**、新着でエンゲージ指標は出さない/0 を隠す。
- **直接構築可**: 累計再生数順 ／ 新着順 ／ クール絞り（現行季）。正確な期間デルタ・個人のアーリー度系は別プロジェクト（視聴履歴拡張）＝スコープ外。

## 新着の2種（データ注記）

REQ-0010 は 2 系統。ビルド時に nvapi series（各話）／snapshot を束ねて両方を静的 JSON 化。

| 種別 | 定義 | カード単位 | 配置 |
|------|------|-----------|------|
| **新着シリーズ** | シリーズ初出日（各話 `startTime` の最小）が新しい | シリーズ | トップ／一覧(新着順) |
| **最新の動画** | 話の `startTime` 最大（既存作の新着話を含む） | 話 | トップ |

## REQ 対応（再掲）

- トップ: 検索/TOP10「すべて見る」=REQ-0001、Hot/人気TOP=REQ-0002、今期/クールから探す=REQ-0004、タグから探す=REQ-0008、新着/最近追加（シリーズ＋最新話）=REQ-0010、ダーク/ライト切替=REQ-0012。
- 一覧: ブラウズ/検索=REQ-0001、勢い/累計=REQ-0002、タグ（フラット・キュレーション正規化含む）=REQ-0003/0008、クール=REQ-0004、新着=REQ-0010、五十音=REQ-0011、♥お気に入り/✓見た=REQ-0015。
- 作品詳細: 各話一覧/概要=REQ-0001、deep-link=REQ-0005、関連シリーズ/続編=REQ-0014、♥/✓=REQ-0015、タグチップ→タグ絞り一覧の回遊=REQ-0003/0008、配信情報欠損は empty 表示。
